# 一站式赴约小管家 UI 设计文档 v1

## 1. 设计目标

这次改版的重点不是“做一个更花的页面”，而是把产品从现在这种：

- 阴暗
- 压抑
- 后台面板感
- 功能大于情绪
- 旅游/活动平台气质不够

升级成一种更适合“赴约”主题的体验：

- 明亮
- 轻盈
- 浪漫
- 有期待感
- 像高质量旅行规划产品，而不是深色工具后台

## 2. 设计结论

### 2.1 视觉总方向

采用：

`明亮奶油底 + 柔和粉紫渐变 + 小面积高饱和点缀 + 大圆角白卡片 + 轻阴影`

而不是：

- 深色夜店风
- 大面积紫黑背景
- 重玻璃拟态
- 到处都发光的赛博感

### 2.2 气质关键词

- 温柔
- 轻松
- 精致
- 期待
- 被照顾
- 像有人陪你准备这场见面

### 2.3 禁止继续出现的气质

- 阴暗
- 压迫
- 冷酷
- 工具后台
- 战术报告
- 低成本“炫彩玻璃”感

## 3. 参考站拆解

本次首页和后续页面，主要参考以下几类产品思路：

### 3.1 Trip.com

参考点：

- 首页首屏结构清晰，用户一眼知道“从哪里开始”
- 搜索 / 规划面板非常强
- 内容多，但层次不乱
- 服务感和可信赖感很强

我们可借鉴：

- 首页必须有一个很明确的“立即开始规划”入口
- 信息结构要像平台，而不是像艺术海报
- 模块节奏要有主次：主入口 -> 推荐路径 -> 场景分类 -> 信任理由 -> 最近内容

### 3.2 Wanderlog

参考点：

- 白色背景和绿色强调，整体很干净
- 规划视图强调秩序，不强调炫技
- 很适合“计划 / 行程 / 手册”类产品

我们可借鉴：

- 白底和浅色卡片是正确方向
- 赴约手册页应该有“文档感”和“计划感”
- 时间线和清单必须清晰，不要做成艺术装饰

### 3.3 Bandsintown

参考点：

- 活动类内容天然适合卡片式浏览
- 视觉上更偏“演出发现”和“附近活动”
- 情绪更活泼、内容更直观

我们可借鉴：

- 场景入口卡片要有活动平台的明快感
- 演唱会 / 音乐节首页模块可以更像“活动发现页”

### 3.4 Wonderplan / AI 旅行规划产品

参考点：

- AI 入口不能复杂
- Prompt 式引导可以降低用户理解门槛
- “告诉我你要去哪里，我来帮你安排”的感觉很重要

我们可借鉴：

- 首页要有“不会填也能开始”的入口
- 创建赴约计划时要有更温和的引导

## 4. 设计原则

## 4.1 明亮优先

背景以浅色、暖白、奶油白为主。  
不再使用整屏深紫、深蓝、黑底作为主背景。

## 4.2 内容优先

页面首先要让用户一眼看清：

- 这是什么产品
- 我能干什么
- 从哪里开始

不是先被风格压到，再慢慢猜功能。

## 4.3 浪漫，但不幼稚

“浪漫”不等于：

- 满屏爱心
- 过度粉色
- 少女贴纸堆砌

而是：

- 柔和渐变
- 轻盈留白
- 带情绪的文案
- 精致而克制的点缀

## 4.4 活泼，但不杂乱

每个模块可以有自己的颜色性格，但不能：

- 每张卡一个完全不同体系
- 颜色互相打架
- 让首页像活动海报拼贴墙

## 4.5 信息架构要像成熟平台

结构必须像成熟旅游/规划平台：

1. 主叙事
2. 快速开始
3. 场景选择
4. 为什么值得用
5. 最近内容 / 历史内容

## 5. 推荐配色系统

## 5.1 主基调

推荐采用：

- 背景色：`#FFF9F0`
- 次级背景：`#FFF5F7`
- 白卡片：`#FFFFFF`
- 主文字：`#1E1E2A`
- 次文字：`#6F6B7D`

## 5.2 主点缀色

### 浪漫粉紫

- `#FFB7C5`
- `#C3A0FF`

用途：

- Hero 渐变
- 重点按钮
- 标签
- 高光插画

### 活力橙黄

- `#FF9F4A`
- `#FFD966`

用途：

- 球赛模块
- CTA 强按钮
- 提醒状态

### 清新薄荷

- `#5DC9A0`
- `#B8E1FF`

用途：

- 通用辅助色
- 安心感模块
- 成功状态、完成感、轻松感

## 5.3 配色比例建议

- 70%：暖白 / 奶油白 / 浅底
- 20%：浅粉紫 / 浅橙 / 浅薄荷背景块
- 10%：高饱和点缀色（按钮、标签、图标、提示）

核心原则：

- 高饱和颜色只做点缀，不做整屏底色
- 主页面以白、暖白、浅粉为基底

## 6. 页面视觉语言

## 6.1 背景

推荐：

- 浅奶油渐变
- 很轻的淡粉或淡紫色光晕
- 局部可加柔焦彩带、演唱会灯海、城市出行图片

不推荐：

- 整屏深色照片蒙版
- 大面积黑紫背景
- 强烈玻璃拟态覆盖整页

## 6.2 卡片

卡片统一规则：

- 背景：白色或非常浅的暖色
- 圆角：`20px - 28px`
- 阴影：柔和浅阴影
- 边框：很淡的浅灰或浅彩色边

不要做成：

- 厚重发亮玻璃卡
- 边缘很硬的深色面板

## 6.3 按钮

主按钮：

- 玫瑰粉 / 活力橙
- 有轻微渐变
- 圆角大
- 文本颜色深

次按钮：

- 白底
- 浅边框
- Hover 时边框和背景稍微染色

## 6.4 图标与装饰

可加入：

- 气球
- 音符
- 星点
- 车票
- 小地图
- 票根
- 足球 / 围巾 / 应援棒等轻线条元素

形式建议：

- 线条插画
- 小图标
- 浅色悬浮元素

不要用：

- 过重的 3D 插画
- 低质 AI 装饰素材

## 7. 首页信息架构

## 7.1 首页目标

首页不是用来“展示所有功能”的。  
首页只解决 3 件事：

1. 让用户知道这个产品是干嘛的
2. 让用户愿意开始创建赴约计划
3. 让用户相信这不是一个杂乱小工具，而是一套认真可用的安排产品

## 7.2 推荐首页结构

### 模块 1：顶部导航

内容：

- Logo
- 产品名
- 我的安排
- 创建赴约计划

风格：

- 白底或半透明浅底
- 轻量悬浮

### 模块 2：Hero 首屏

左侧：

- 一句清晰的主标题
- 一句温柔的副标题
- 3 个价值点标签

右侧：

- 一个强引导规划卡
- 像旅游平台的搜索框区域，但换成“赴约规划输入卡”

### 模块 3：快速问题入口

用 3-4 个 prompt 卡片告诉用户：

- 第一次一个人去演唱会怎么办
- 散场赶高铁怎么安排
- 音乐节怎么准备才不狼狈

### 模块 4：场景入口

3 个主入口：

- 演唱会赴约
- 音乐节赴约
- 球赛赴约

### 模块 5：为什么值得用

3 张卡片说明：

- 不只是攻略
- 不只是票务
- 不只是提醒

而是“陪你认真安排这次赴约”

### 模块 6：最近安排 / 历史手册

如果有历史内容，就展示最近 3 条  
如果没有，就用友好的空状态

## 8. 创建页设计规范

创建页应该像“温柔的规划表单”，不是后台录入表。

## 8.1 整体方向

- 白底
- 表单分组明显
- 字段不宜太密
- 可加入“推荐填写顺序”

## 8.2 表单分区

### 分区 1：这次赴约是什么

- 活动类型
- 活动名称
- 城市
- 场馆
- 日期

### 分区 2：你打算怎么去

- 出发地
- 是否跨城
- 出行偏好
- 预算

### 分区 3：你更在意什么

- 票档 / 座位
- 是否第一次
- 备注

## 8.3 视觉规范

- 输入框底色：浅灰白
- 聚焦描边：粉色或薄荷绿
- 按钮：暖粉或橙色

## 9. 赴约手册页设计规范

## 9.1 整体目标

这页不应像“数据报告”，而应像：

- 一份整理好的赴约手册
- 一份可以收藏和分享的页面
- 一份活动前后都愿意再打开看的安排页

## 9.2 模块结构建议

### 模块 1：封面卡

- 活动名称
- 场馆 / 城市 / 日期
- 一句本次赴约摘要

### 模块 2：赴约安心度

- 不要太像评分系统
- 推荐更像“本次安排状态”

### 模块 3：提前提醒你

- 3-5 条重点提醒
- 每条卡片化
- 可加小图标

### 模块 4：这一天可以这样安排

- 时间线风格
- 白底 + 彩色节点

### 模块 5：出发前收一收

- 清单三列
- 卡片化展示

### 模块 6：给你的贴心提醒

- 到场建议
- 散场建议
- 新手提醒

## 9.3 视觉要求

- 主体区域大面积白卡片
- 彩色只用在模块头、标签、节点、按钮
- 模块之间留白明显

## 10. 分享页设计规范

分享页要比主产品页更像“内容卡”。

重点：

- 看起来适合截图和转发
- 不要有太重后台感
- 适当强化仪式感和浪漫感

建议：

- 顶部一句温柔引导
- 中间直接给出手册主体
- 底部再引导“我也创建一份”

## 11. 动效建议

动效不能多，但要有。

建议加入：

- 首屏轻微上浮进入
- 按钮 hover 微微提起
- 卡片进入时轻微淡入
- 时间线节点渐次出现

不建议：

- 大面积漂浮粒子
- 连续闪烁
- 过度 bounce 动效

## 12. 字体与排版建议

## 12.1 字体倾向

中文优先选择：

- 更圆润一点的现代无衬线
- 不要过硬的系统黑体感

## 12.2 标题层级

- Hero 标题：强、短、好懂
- 卡片标题：16-22px
- 正文：14-16px
- 次说明：12-14px

## 12.3 文案语气

应该像：

- 朋友陪你一起想
- 温柔而有分寸
- 不夸张，不油腻

不应该像：

- 指挥系统
- 报告系统
- 硬核任务工具

## 13. 组件规范

## 13.1 大圆角

统一建议：

- 主卡：`28px`
- 二级卡：`22px`
- 输入框：`16px`
- 按钮：`999px`

## 13.2 阴影

统一建议：

- 很浅
- 偏柔
- 不偏黑

例如：

- `0 12px 30px rgba(80, 60, 120, 0.08)`

## 13.3 边框

统一建议：

- 浅灰 / 浅粉 / 浅薄荷
- 透明度低

## 14. 直接禁止的视觉问题

后续开发中，以下问题不能再出现：

1. 大面积深色背景主导页面
2. 卡片像后台面板
3. 整体颜色脏、灰、闷
4. 炫彩只靠背景图，不靠组件体系
5. 首页没有明确主入口
6. 表单像管理系统
7. 手册页像报告页

## 15. 分阶段落地建议

## Phase 1：先统一首页设计系统

- 改首页背景到浅色体系
- 改 Hero
- 改规划面板
- 改场景卡
- 改价值卡

## Phase 2：创建页重做

- 白底表单
- 分组结构
- 柔和输入框
- 更好的填写引导

## Phase 3：赴约手册页重做

- 文档式、卡片式排版
- 清单和时间线重做
- 分享页同步升级

## 16. 最终推荐的主视觉方向

如果现在只定一个明确方向，我推荐：

`温柔奶油底 + 粉紫渐变高光 + 白卡片 + 少量橙金点缀`

这是最适合“赴约”这个词的首页方向。

它既有：

- 浪漫感
- 安心感
- 女性友好
- 活动氛围感

也保留了：

- 旅行产品的秩序
- AI 规划工具的清晰度
- 平台首页的专业感
