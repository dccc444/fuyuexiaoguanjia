# 赴约小管家 · 网页整体升级方案 v1

文档目标：把当前网站从“局部改样式”升级成“统一的追星赴约产品体验”，后续所有页面改版都按这份执行。

---

## 1. 这次升级要解决什么

当前问题不是单一配色问题，而是 4 个问题叠在一起：

1. 首页和内页不像同一个产品
2. 视觉语言过于依赖粉色，层次单薄
3. 模块加得太多，但信息架构没有先收敛
4. 交互动效还停留在“有没有”，没有建立统一的 motion system

产品定位也必须重新钉牢：

- 它不是泛旅游网站
- 它不是普通表单工具
- 它是 `追星赴约助手 + 观演观赛小管家`
- 核心场景优先级：`演唱会 / 音乐节 > 球赛 > 通用工具`

一句话原则：

**先把“赴约感”和“追现场感”做对，再把“高级感”和“科技感”做精。**

---

## 2. 这次我从参考站学什么

这次不是照抄站点，而是提炼它们各自最有价值的设计能力。

### 2.1 Explora Journeys

要学的不是“豪华邮轮感”，而是：

- 大幅留白下的信息秩序
- 宽屏沉浸封面的节奏控制
- 图像、文案、CTA 之间的呼吸感
- 复杂内容仍然能维持优雅层级

可迁移到赴约小管家：

- 首页与手册页的 hero 不再做成“大色块 + 一堆标签”
- 大图只负责定氛围，不负责堆信息
- 重要信息要做成“一眼可扫”的摘要带

### 2.2 Trip.com

要学的是：

- 一级入口极其清晰
- 大量模块并存时仍有明确主次
- 搜索 / 选择 / 筛选的效率感
- 场景入口和内容入口并排共存

可迁移到赴约小管家：

- 创建页要更像“选择器驱动的赴约配置器”
- 系统已有的城市、场馆、场景优先直接选
- 大量功能通过侧栏和二级入口组织，而不是全部堆在主表单里

### 2.3 Airbnb

要学的是：

- White card system 的清爽组织力
- 低压力筛选器和标签系统
- 大留白下的高扫描效率
- 轻内容也可以很精致

可迁移到赴约小管家：

- 我的安排、分享页、场馆规则页要回到更清晰的卡片系统
- 标签筛选、城市选择、场馆选择优先 chip 化

### 2.4 Layla

要学的是：

- AI 助手的“陪伴式语气”
- 情绪图像与工具效率并存
- 规划流程不是冷冰冰的录入，而是引导式共创

可迁移到赴约小管家：

- 创建页不是后台录入页
- 手册页不是报告页
- 所有 CTA 和提示语都要更像“小管家在陪你安排”

## 3. 专业术语库，后面统一按这个说

这份术语库用于后续页面评审、设计文档、开发标注，避免“好看一点”“高级一点”这种模糊描述。

### 3.1 视觉分层术语

- `Elevation`：通过阴影与明暗差建立层级抬升
- `Negative Space`：通过留白分组，而不是通过加更多边框分组
- `Grid Layout`：统一内容栅格，保证页面稳定
- `Divider Hierarchy`：主区块、次区块、轻分组使用不同强度的分割线
- `Card Nesting`：只在必要时使用卡片内嵌，不允许无意义层层套卡
- `Padding Hierarchy`：页面、模块、组件、字段必须有不同等级的内边距
- `Overlay`：用浅蒙层做分区叠色，不用大面积纯色硬切
- `Subtle Texture`：极浅纹理底，避免大面积纯底过于空和塑料感

### 3.2 色彩系统术语

- `Primary / Secondary / Accent`：主色、辅色、点缀色职责分离
- `Neutral Scale`：文本与背景使用独立的中性色阶
- `Functional Color`：成功、风险、提醒、信息必须有固定功能色
- `Value Hierarchy`：同色系通过明度差区分信息层级
- `Desaturated Tone`：以低饱和大底为主，高饱和只做局部强调
- `Color Role Lock`：一旦确定页面 accent，不允许中途乱切第二主色

### 3.3 交互动效术语

- `Pressed Offset`：点击时轻微下沉
- `Spring Scale`：按钮或卡片用弹簧缩放反馈
- `State Layer`：hover / press 用半透明状态层，而不是突兀换整块色
- `Sticky Pin`：步骤导航、目录、筛选条滚动吸顶
- `Step Form Wizard`：分步式表单，不让用户一次面对整张长表
- `Auto Complete Dropdown`：输入联想下拉
- `Chip Filter`：标签式筛选
- `Mount Transition`：模块入场的淡入 + 小幅位移
- `Scroll Anchor`：点击目录跳转对应区块
- `Accordion`：长说明折叠展开
- `Toast`：轻提示，不打断操作
- `Micro Feedback`：操作后的细微成功反馈
- `Number Ticker`：数值滚动到目标值
- `Parallax Layer`：轻视差层，不是强烈滚动特效
- `Spotlight Cursor`：鼠标探照灯，仅首页或品牌页可用，内页慎用

## 4. 新的视觉方向，不再纯粉

这次不再走“满页粉白浪漫”，而是走：

**明亮基底 + 多层中性色 + 一条暖情绪主线 + 一条冷科技辅线**

### 4.1 推荐主路线

推荐方案名：

`Aurora Fan Date System`

核心气质：

- 明亮
- 轻盈
- 有追现场氛围
- 有现代科技感
- 不甜腻

### 4.2 配色结构

#### 中性基底

- `Cloud White`：#F7F7FA
- `Soft Paper`：#FFFFFF
- `Mist Gray`：#E9EAF0
- `Ink Gray`：#2B2F38
- `Muted Copy`：#6C7280

#### 情绪主线

- `Rose Signal`：#E66B7A
- `Champagne Glow`：#E7C27D

#### 科技辅线

- `Arena Blue`：#5B6CFF
- `Mint Pulse`：#7BC8B4

### 4.3 配色使用规则

- 70%：中性浅底和白卡
- 20%：分区淡色和蒙层
- 10%：强强调色

禁用：

- 整页纯粉
- 深色整图压底
- 紫蓝霓虹全页泛光
- 每个模块都用不同高饱和颜色抢戏

### 4.4 颜色落位

- `Rose Signal`：主 CTA、选中 chip、重点提醒
- `Champagne Glow`：封面光晕、数字亮点、轻高光
- `Arena Blue`：导航当前态、链接、规则与信息型模块
- `Mint Pulse`：完成态、安心感、轻成功反馈

## 5. 动效系统，不是乱飞

动效要有“系统”，而不是想到哪加到哪。

### 5.1 动效层级

#### A. 品牌层

只允许出现在首页 / 开屏 / 分享封面：

- 中心圆形遮罩收缩开屏
- 标题字距收拢入场
- Hero 图片轻视差
- 鼠标探照灯
- 玻璃分层卡片视差

#### B. 页面层

出现在创建页 / 手册页 / 我的安排：

- 顶部封面 mount transition
- sticky 步骤导航
- 区块渐显
- 抽屉式补充信息面板
- Accordion 展开

#### C. 组件层

出现在所有交互里：

- pressed offset
- focus highlight
- state layer hover
- chip filter 切换
- toast
- 数值滚动

### 5.2 推荐时长

参考 Material motion 和产品型界面节奏：

- 微交互：150ms - 220ms
- 表单状态变化：180ms - 260ms
- 抽屉 / 弹层：240ms - 320ms
- 主页品牌级入场：400ms - 650ms

### 5.3 推荐组合

#### 首页

- `圆形遮罩收缩 + 标题字距收拢 + CTA 弹簧入场 + 背景轻光晕漂移`

#### 创建页

- `hero 轻淡入 + 左侧进度 sticky + step section mount transition + chip press`

#### 手册页

- `顶部摘要卡浮现 + 安心度数字 ticker + 时间线分段 reveal + 风险卡 hover expand`

#### 分享页

- `封面先入场 + 关键标签后入场 + CTA 上浮`

禁用：

- 全页每个模块都 fade-up
- 长时间加载动画
- 剧烈弹跳
- 无意义粒子满屏乱飘

## 6. 信息架构升级，按功能模块重新收口

这部分严格按 `功能模块.docx` 组织，不再按“旅游产品逻辑”展开。

## 6.1 顶层导航

建议顶层只保留：

1. 首页
2. 创建计划
3. 我的安排

辅助入口：

- 分享页不进主导航
- 记账本 / 设置在用户侧入口中

## 6.2 首页结构

首页不是功能大全，而是明确的分流页。

### 首页一级区块

1. Hero：一句价值主张 + 两个主入口
2. 场景分流
   - 演唱会 / 音乐节赴约
   - 球赛赴约
3. 高频能力摘要
   - 场馆规则
   - 出片安排
   - 搭子 / 物料
   - 散场返程
4. 最近安排 / 历史入口

### 首页侧栏建议

按你功能模块拆成两大板块：

- `演唱会 / 音乐节`
  - 衣
  - 食
  - 住
  - 行
  - 票
  - 社交 / 物料

- `球赛`
  - 行程规划
  - 信息模块
  - 现场观赛指南
  - 社交与情绪宣泄
  - 应援与物料

## 7. 内页升级方案，按页面逐个定义

## 7.1 创建页

目标：从“长表单”升级成“赴约配置器”。

### 创建页结构

1. 压缩版顶部封面
   - 只保留一句主标题
   - 一句副文案
   - 3 到 4 个当前场景关键词

2. 左侧轻步骤栏
   - 只保留步骤名 + 进度
   - 不放大段解释
   - sticky pin

3. 主表单区分 3 步
   - `先定活动和场馆`
   - `再定出行和预算`
   - `最后补充追现场偏好`

4. 规则预览区
   - 命中场馆后显示一张摘要卡
   - 默认摘要，详情 Accordion 展开

### 创建页必须落地的交互

- 热门城市 Quick Pick
- 场馆联想下拉
- 场馆卡片直选
- chip filter 选追星偏好
- blur validate
- submit feedback
- sticky progress

### 创建页不要再有的东西

- 重复的解释卡
- 满屏说明文案
- 深色背景大图
- 第二套与首页无关的设计语言

## 7.2 赴约手册页

目标：从“AI 结果页”升级成“可带着出门的赴约手册”。

### 手册页结构

1. 封面摘要
   - 活动 / 艺人 / 场馆 / 日期
   - 安心度
   - 这次小目标

2. 今天这样去最顺
   - 时间线
   - 入场节奏
   - 返程节点

3. 提前提醒你
   - 风险卡
   - 规则卡
   - 禁带 / 可带

4. 追现场专项模块
   - 穿搭和妆造提醒
   - 物料与会合提醒
   - 票档 / 看台提醒
   - 散场返程提醒

5. 可折叠扩展
   - 场馆来源
   - 交通详情
   - 住宿 / 餐饮补充

### 手册页动效

- 数值滚动
- 时间线分段 reveal
- hover expand tip
- Accordion
- toast 复制分享链接

## 7.3 我的安排

目标：从列表页升级成“赴约档案页”。

### 模块

- 当前安排
- 历史安排
- 收藏场馆
- 快速复用上次配置

### 交互

- card hover expand
- chip filter
- sort / duplicate / share

## 7.4 分享页

目标：从“普通详情分享”升级成“可转发的赴约邀请卡”。

### 模块

- 封面图 / 氛围图
- 这次去哪 / 看谁
- 核心摘要
- 一个明确 CTA

分享页不承载所有细节，重点是：

- 好转发
- 好看
- 一眼看懂

## 9. 升级顺序，先做什么

这次不建议全站一起乱改。

### Phase A：先立系统

1. 统一颜色 token
2. 统一字体层级
3. 统一圆角与阴影
4. 统一按钮、chip、输入框、卡片规范
5. 统一 motion token

### Phase B：重做核心链路

1. 创建页
2. 赴约手册页
3. 我的安排
4. 分享页

### Phase C：补功能模块入口

1. 首页双板块侧栏
2. 演唱会 / 音乐节专区入口
3. 球赛专区入口

### Phase D：先落最有辨识度的追星功能

1. 穿搭 / 妆造
2. 物料 / 搭子 / 会合
3. 场馆规则
4. 散场返程
5. 票档 / 看台提醒

### Phase E：再铺深功能

1. 食住行票专项页
2. 社区与物料互换
3. 记账和 AA 分账
4. 设置和偏好

---

## 10. 这次升级后的审美红线

后续所有页面都必须避开这些问题：

1. 不能再靠一整页粉色制造浪漫
2. 不能再用深色整图压背景制造高级感
3. 不能再给每一屏堆满解释性板块
4. 不能让首页和内页像两个产品
5. 不能把“赴约助手”做成“普通旅行工具”
6. 不能让功能模块漂离 `功能模块.docx` 本身

---

## 11. 接下来怎么执行

我建议后续执行顺序固定成这样：

1. 先按这份方案重做 `创建页`
2. 创建页过了，再重做 `赴约手册页`
3. 再统一 `我的安排` 和 `分享页`
4. 最后按功能模块逐块接功能

这样我们就不会再陷入“边改边漂，越改越散”。
