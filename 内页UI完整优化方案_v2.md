# 赴约小管家 · 全量内页 UI 完整优化方案 V2

状态：重学后修订版  
用途：替代此前偏“粉色堆叠”和“浅色大白板”的方案，作为后续所有内页改版的唯一基准。  
范围：创建页、赴约手册页、我的安排页、分享页、后续演唱会/音乐节/球赛专区页。  

---

## 0. 先说结论

你前面骂得没错，之前那几版确实有明显问题：

- 配色太“自我感动”，不够产品化
- 结构像在拼情绪卡片，不像成熟网站
- 缺少真正的主操作区
- 视觉层次主要靠浅粉渐变，不靠信息架构
- 太想做“浪漫”，结果做成了“轻飘、幼、糊”

这版方案的核心修正是：

`不再用粉色当设计本身，而是把粉色降级成品牌点缀。`

真正要学的是你给的这些优秀站点背后的共同能力：

1. 清晰的操作主轴
2. 强搜索/强筛选/强导航
3. 图片与内容并存，但图片不压死功能
4. 白底或浅底下，依然有非常强的层次感
5. 局部高饱和，不是全页高饱和

---

## 1. 这次到底学了什么

这次不是“看几个截图说灵感”，而是把参考站拆成了可执行的产品语言。

### 1.1 携程酒店页

从你给的截图里，真正值得学的不是蓝色，而是这 5 件事：

1. 左侧导航是稳定骨架  
   页面一打开，用户知道自己在哪个系统、在哪个一级栏目。

2. 中央搜索面板是绝对主角  
   用户第一眼就知道“我要在这里开始”。

3. 右侧大图承担“情绪”和“城市目的地感”  
   不是用来盖住内容，而是给页面建立视觉锚点。

4. 信息密度高，但区域职责非常明确  
   搜索区、活动区、推荐区、内容区各自有边界。

5. 局部强色非常果断  
   蓝就是主操作色，白就是内容底，绿色图片是辅助，不混。

### 1.2 马蜂窝订酒店页

这个页面更值得学的是“攻略感”和“留白纪律”。

值得学的点：

1. 大面积白底  
   不是空，而是给重点信息留舞台。

2. 橙色只用于关键动作和重点 tab  
   很克制，所以显得干净。

3. 图片卡直接承担内容价值  
   不是装饰图，而是“主题住宿”的入口。

4. 结构非常稳定  
   搜索、卖点、主题、内容，一层一层展开。

### 1.3 Tripadvisor

值得学的是“内容流”。

1. 搜索入口简单
2. 大图承担灵感
3. 内容卡片形成连续浏览节奏
4. 用户会愿意继续往下刷

这对 `我的安排页` 和 `分享页` 很重要。

### 1.4 Trip.com

值得学的是“国际化成熟产品的密度控制”。

1. 顶部导航非常完整，但不显乱
2. 搜索面板结构极稳
3. 卡片系统统一
4. 强活动视觉与功能面板并存

这意味着：

`赴约小管家` 后续不能只做“几张浅色卡片”，而是要建立真正的模块化内容系统。

### 1.5 Wanderlog / TripIt / Mindtrip / Wonderplan

这几类产品共同给出的结论是：

1. 规划类产品必须让用户“快速开始”
2. 不要逼用户写长表单
3. 一定要有“结构化摘要”
4. 协作、分享、复制、再生成都要顺手

来源：

- [Wanderlog](https://wanderlog.com/)
- [TripIt](https://www.tripit.com/web)
- [Mindtrip](https://mindtrip.ai/)
- [Wonderplan](https://wonderplan.ai/)

## 2. 重新定义产品的内页气质

### 2.1 不再是“粉色浪漫页”

浪漫不是：

- 整页浅粉
- 字也粉、边框也粉、背景也粉
- 所有模块都雾蒙蒙

那样只会显得：

- 幼
- 糊
- 没主次
- 不专业

### 2.2 新气质定义

内页应该是：

`白底的成熟规划平台 + 温柔的品牌情绪 + 少量追现场的热度`

关键词：

- 清楚
- 轻快
- 高级
- 不装嫩
- 有攻略感
- 有助手感

---

## 3. 配色系统 V2

### 3.1 总原则

以前的问题是：

- 主色太多粉
- 中性色不够硬
- 冷暖没有明确分工

现在改成三层配色结构：

1. `基础中性色`
2. `操作功能色`
3. `情绪点缀色`

### 3.2 基础中性色

这些颜色决定页面是不是成熟。

- 页面底色：`#F7F8FA`
- 主内容卡：`#FFFFFF`
- 次级区块底：`#F4F6F8`
- 边框线：`#E5EAF0`
- 轻分割线：`#EEF2F6`

文字：

- H1/H2：`#1F2A37`
- 正文：`#344054`
- 次正文：`#667085`
- 辅助字：`#98A2B3`
- 占位符：`#B3BDC8`

这套中性色要占页面的 70% 以上。

### 3.3 操作功能色

这次真正要学 OTA。

建议：

- 主操作色：`#2F80ED`
- 悬浮/active：`#1C64D1`
- AI 辅助色：`#6C8CFF`
- 功能绿色：`#2FB67C`
- 提醒橙色：`#F59E0B`
- 危险红：`#E65F5C`

用途：

- 搜索 / 生成 / 确认
- 选中态
- 状态标签

### 3.4 品牌情绪色

不是主力色，只是点缀。

- 珊瑚粉：`#F08A93`
- 杏桃色：`#F6B37D`
- 浅金色：`#F4D27A`
- 雾粉紫：`#E7DDF7`

只能用于：

- 小面积标签
- 局部渐变
- 数字、badge、标题强调

不能用于：

- 全页面底
- 全模块底
- 长正文背景

### 3.5 图片色策略

以后页面里的图片色不能乱。

优先选：

- 蓝天 / 海景 / 城市夜景 / 场馆灯光
- 白、蓝、绿、金这些容易和浅底融合的图

避免：

- 高饱和紫粉大片压满整个页面
- 复杂彩带大图铺满内页背景

## 4. 结构系统重做

你之前说我结构傻，这个判断对，根本原因是我没有建立“强主区”。

### 4.1 以后每个内页都必须有主区

每页必须有：

1. `导航骨架`
2. `主操作区`
3. `内容区`
4. `辅助区`

不能再：

- 进来就是一堆视觉卡片
- 每块都差不多重要
- 没有入口重心

### 4.2 统一布局母版

内页统一采用：

- 顶部全局导航
- 左侧轻导航 / 目录 / 进度
- 中央主内容
- 右侧根据页面类型决定是否出现辅助内容

这是从携程和 Trip.com 学来的，不是照抄，是学它们的“稳定骨架”。

## 5. 页面逐一优化方向

## 5.1 创建计划页

### 当前问题

- 看起来还是“美化过的表单”
- 没有像成熟旅行产品那样的强搜索感
- 粉色太多，字段区不够利落

### 新方向

创建页应该像：

`酒店/行程搜索面板 + AI 偏好配置面板 + 赴约情绪导入`

### 新结构

#### A. 顶部轻 Hero

作用：

- 说明“这是在安排一场赴约”
- 不占太高
- 不重复叙事

#### B. 主搜索面板

第一屏最核心区域，像携程/Trip.com 一样处理：

- 活动类型
- 活动名称 / 艺人
- 城市
- 场馆
- 日期
- 时间

要求：

- 用横向主面板组织
- 主 CTA 明确
- 视觉上最像“开始点”

#### C. 偏好配置区

第二层做：

- 同行人数
- 预算倾向
- 出行方式
- 最在意什么
- 是否需要住宿

#### D. 追星重点区

第三层才做：

- 衣
- 食
- 住
- 行
- 票
- 社交

要求：

- 不要再全是输入框
- 多用 choice chips、segment control、pick cards

#### E. 场馆规则预览区

作为强辅助区，像攻略摘要卡，不像警告面板。

### 视觉方向

- 主面板可以用蓝白结构
- 品牌粉只做 badge / 点缀 / 小标签
- 字体更硬朗一点

---

## 5.2 赴约手册页

### 当前问题

- 一度被做成深色 dashboard
- 浅底版又缺乏真正的攻略结构
- 模块卡太均质

### 新方向

手册页应该像：

`TripIt 的 itinerary summary + Tripadvisor 的内容流 + 追星助手的现场提醒`

### 新结构

#### A. 头部总览

像 itinerary header：

- 活动名称
- 艺人 / 主队
- 城市 / 场馆
- 日期 / 时间
- 赴约安心度
- 复制分享

#### B. 六大模块摘要

必须明确作为主结构出现：

1. 衣
2. 食
3. 住
4. 行
5. 票
6. 社交

每个模块不是同款卡，而是：

- 有不同的小图标
- 有不同的功能色线
- 有不同的摘要类型

#### C. 规则与提醒

- 场馆规则
- 风险提醒
- 入场清单

这里应该更像攻略说明块，不是情绪卡。

#### D. 当天路线与时间线

- 到场
- 入场
- 会合
- 观演
- 散场
- 返程

### 视觉方向

- 白底主手册
- 卡片分层更像内容产品
- 少量蓝、绿、橙、粉区分模块
- 不做一排发灰渐变卡

---

## 5.3 我的安排页

### 当前问题

- 太像普通列表
- 没有灵感和管理并存的感觉

### 新方向

像：

`旅行平台里的我的行程 + 内容推荐流`

### 结构

#### A. 顶部筛选区

- 城市
- 场景
- 时间
- 状态

#### B. 我的赴约卡片流

每张卡至少有：

- 活动名
- 艺人/主队
- 场馆
- 时间
- 安心度
- 快捷动作

#### C. 再次安排/相似推荐

这个区可以学 Tripadvisor 的内容流：

- 相似城市场馆
- 相似场景灵感

---

## 5.4 分享页

### 当前问题

- 像结果页复印件

### 新方向

像：

`可转发的赴约摘要页`

### 结构

1. 顶部一句话概览
2. 活动核心信息卡
3. 六大模块精简摘要
4. CTA：我也生成一份

要求：

- 比手册页更轻
- 比手册页更适合截图和转发

## 6. `衣食住行票社交` 的真正页面化方式

之前的问题是把这些功能当成“文案块”，这不够。

现在明确它们的 UI 形式。

### 衣

- 形式：搭配建议卡 + 妆造提醒 chip + 天气补位提示
- 颜色：雾粉 + 轻金

### 食

- 形式：入场前 / 场内 / 散场后三段式卡片
- 颜色：浅橙 + 暖黄

### 住

- 形式：住哪里最顺的策略卡
- 颜色：薄荷绿 + 灰蓝

### 行

- 形式：时间线 + 风险标签 + 返程卡
- 颜色：蓝色系

### 票

- 形式：票档摘要、分区提醒、入场口提示
- 颜色：深蓝 + 浅灰

### 社交

- 形式：搭子会合卡、物料提醒卡、一个人去也不慌卡
- 颜色：珊瑚粉 + 杏色

## 7. 交互语言

这次要真正用成熟产品的交互，不再只靠视觉。

建议统一采用：

- `Chip Filter`
- `Step Form Wizard`
- `Auto Complete Dropdown`
- `Sticky Pin`
- `Accordion`
- `Tooltip Popover`
- `Mount Transition`
- `Pressed Offset`

不建议现在上：

- 全局探照灯
- 大粒子背景
- 复杂 3D 玻璃拟态
- 持续炫技 hover

---

## 8. 图片策略

你给的参考里有一个非常关键的共性：

`图片是内容，不是墙纸。`

以后内页图片这么用：

1. 创建页  
   只在顶部或局部侧卡放情绪图

2. 手册页  
   只在摘要、分享、推荐区域放图

3. 我的安排  
   卡片封面图可以增强内容感

4. 分享页  
   可以强化可截图的首屏视觉

禁止：

- 整页用复杂活动图做背景
- 用高噪点图压正文区

## 9. 这次修订后的总设计原则

### 9.1 配色原则

- 白底/浅底做主场
- 蓝做操作色
- 粉做品牌情绪点缀
- 绿/橙做模块辅助色
- 中性色撑起成熟感

### 9.2 结构原则

- 先有操作区，再有内容区
- 先让用户知道去哪点，再讲情绪
- 模块必须有主次关系

### 9.3 内容原则

- 围绕 `衣食住行票社交`
- 围绕真实赴约路径
- 围绕“能不能帮用户少慌一点”

### 9.4 气质原则

- 不幼
- 不糊
- 不飘
- 不后台
- 不旅游 OTA 复制

1. 先重做创建页母版

## 12. 最终结论

这版文档之后，内页不再走“粉色情绪页”路线，而改成：

`成熟旅行产品的信息秩序 + 追星助手的情绪表达 + AI 规划工具的高效入口`

如果按这版做，页面会更像：

- 一个真正能用的产品
- 一个有品牌感的追星赴约助手

而不是一页页“好像在设计，但其实没主轴”的卡片拼贴。
