# 一站式赴约小管家 - 待办事项清单 (MVP 补齐与进阶)

基于产品规划文档（`一站式赴约小管家_完整产品开发规划文档.docx`）与当前代码库的实现进度，梳理出以下待办事项。这些事项是确保产品达到完整 MVP 验收标准以及后续进阶所必须完成的功能。

## 🔴 P0 优先级：MVP 核心闭环补齐（必须完成）

### 1. 赴约成功率评分机制
当前 AI 生成了内容，但缺少直观的评分和量化的提分指导。
- [ ] **后端 (AI Prompt)**：修改 `generator.js`，强制要求 AI 输出一个 0-100 的“赴约成功率评分”。
- [ ] **后端 (AI Prompt)**：要求 AI 输出 1-2 条具体的“提分建议”（例如：“提前 60 分钟到场，评分可提升至 88 分”）。
- [ ] **前端 (UI)**：在 `BattleBookView` 顶部增加醒目的评分展示（如环形进度条或雷达图），并展示提分建议。

### 2. 现场指挥中心（活动当天行动面板）
目前仅有静态列表，缺乏活动当天的临场感。
- [ ] **前端 (交互)**：在手册详情页增加“切换到当天模式”的入口。
- [ ] **前端 (UI)**：开发“现场指挥中心”视图，包含：
  - 动态倒计时（距离入场、距离散场）。
  - 可勾选的入场前 Checklist（证件、门票、充电宝等）。
  - 天气与补给提醒卡片。
- [ ] **后端 (数据)**：确保 AI 返回的数据结构支持 CheckList 渲染。

### 3. 用户反馈与纠错机制
MVP 验收标准明确要求用户能对不准确的规则进行反馈。
- [ ] **前端 (UI)**：在场馆规则模块或页面底部增加“我要纠错 / 规则不准”按钮。
- [ ] **前端 (表单)**：实现一个轻量级的反馈弹窗（收集纠错内容）。
- [ ] **后端 (API)**：新增 `/api/feedbacks` 接口，接收并存储用户的纠错数据。

### 4. 核心场馆规则库扩充
当前演示数据不足，需要覆盖重点城市。
- [ ] **数据录入**：在 `venue-rules.js`（或数据库中）补充以下核心场馆的规则：
  - 北京：鸟巢、华熙、工体
  - 上海：梅奔、虹口、上体
  - 杭州：奥体
  - 成都：凤凰山
  - 广深：1-2个热门场馆

## 🟡 P1 优先级：产品体验增强（建议实现）

### 1. 票务助手模块
解决用户购票、补票和避坑的痛点。
- [ ] **前端/后端**：在创建表单中增加“是否已购票”的选项。
- [ ] **后端 (AI)**：如果用户未购票，AI 应输出票务建议（如抢票策略、二手票防骗指南、实名制/纸质票说明）。

### 2. 多场景深度定制（追星专属/球赛专属）
当前三种场景的差异化不够明显。
- [ ] **前端 (UI)**：为演唱会、音乐节、球赛设计差异化的主题色或背景图。
- [ ] **后端 (逻辑)**：细化 AI 生成逻辑，针对演唱会增加“应援物/穿搭建议”，针对球赛增加“主客队座位区防冲突提醒”。

### 3. 后台管理系统 (Admin)
用于处理用户纠错和维护场馆数据。
- [ ] **后端/前端**：搭建一个简单的运营后台界面（或接口）。
- [ ] **功能**：支持管理员查看用户反馈、修改和发布新的场馆规则。

## 🟢 P2 优先级：进阶功能（后续迭代）

### 1. 同场情报与社交（轻量级）
- [ ] **前端/后端**：实现轻量级的同场社交面板（留言板形式）。
- [ ] **功能**：支持发布“找搭子”、“拼车”、“拼房”、“物料互换”等标签信息。

### 2. 用户体系体系与个人中心
- [ ] **后端**：接入微信登录或手机号登录（替代目前的游客态 Token 模式）。
- [ ] **前端**：完善“我的主页”，支持设置粉籍、主队偏好、预算偏好等长期配置。

### 3. 微信小程序化
- [ ] **工程**：将现有的 H5 逻辑迁移至 Taro 或原生微信小程序框架。
- [ ] **功能**：接入微信分享卡片定制、订阅消息提醒（如开票提醒、出发提醒）。
