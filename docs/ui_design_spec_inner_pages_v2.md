# 一站式赴约小管家 · 内页 UI 设计文档

版本：V2.0 明亮浪漫版  
文档状态：定稿  
设计目标：彻底去除阴暗、沉重、旧后台式的内页感受，打造通透、温暖、治愈、陪伴式的赴约规划体验，让每一场期待都像被晨光、花瓣和温柔提醒轻轻托住。

---

## 一、先明确本轮要纠正的问题

当前内页的核心问题，不是信息缺失，而是视觉气质完全跑偏：

- 使用了大面积、整页铺满的深色现场照片作为主背景
- 背景构图过暗，压住了页面主体内容
- 卡片虽然是浅色，但被阴暗背景包围后，整体仍然显得闷、脏、重
- 视觉更像“活动后台系统”或“夜场海报”，不像“赴约小管家”
- 表单区仍然有明显填表感，不够像一位贴心的追星助手在陪你安排

本轮设计文档的首要目标，就是彻底解决这个问题。

### 1.1 背景使用的硬性规则

从本版开始，内页必须遵守以下规则：

- 禁止使用整页深色演唱会照片作为主背景
- 禁止使用任何大面积暗蓝、暗紫、深灰、黑色摄影图当页面底图
- 页面主背景必须是浅色、暖色、通透的奶油系渐变底
- 如果使用现场照片，只能作为局部情绪装饰
- 照片必须经过提亮、柔化、降对比、裁切处理
- 照片只能出现在局部卡面、顶部情绪横幅、小面积轮播或插图区域
- 照片绝不能比内容卡片更抢视觉重心

一句话原则：

**照片是点缀气氛，不是页面底色。**

---

## 二、设计总则

### 2.1 设计关键词

- 明亮：高亮度、低压迫感、充足留白，拒绝暗底压屏
- 大气：版面开阔、呼吸感强、层级清晰但不过度切割
- 浪漫：暖色、柔光、浅渐变、细腻文案、追星赴约的期待感
- 陪伴：像小管家在陪你安排，而不是系统要求你逐项提交
- 轻盈：圆角、浅阴影、柔边界、轻动效，拒绝厚重块状感

### 2.2 核心理念

> 把每一场期待，提前安排成更安心也更浪漫的赴约。

界面不应像填表系统，而应像一位贴心的追星助手：

- 用选择题代替填空题
- 用陪伴感取代冰冷录入
- 用轻提示代替系统命令
- 用情绪氛围托住功能内容

### 2.3 产品气质定位

这不是旅游攻略站，也不是通用活动工具，而是：

- 演唱会赴约助手
- 音乐节赴约助手
- 球赛赴约助手
- 追星人、球迷、搭子、朋友之间的赴约安排工具

所以内页气质必须更贴近：

- 期待见面前的认真准备
- 想出片、想安心、想顺利、想不踩坑
- 被理解、被陪着安排的感觉

## 三、视觉规范

### 3.1 色彩系统

| 角色 | 色值（HEX） | 用途 |
| --- | --- | --- |
| 主背景 | `#FFF9F5 → #FFF0F0` | 全局背景，由奶油色向淡粉色过渡 |
| 卡片背景 | `#FFFFFF` / `#FFF8F4` | 主卡片、次级区块 |
| 品牌主色 | `#C45C66` | 重点按钮、进度百分比、强调文字 |
| 辅助色 | `#FFCDC2` | 标签背景、选中态、装饰线 |
| 点缀色 | `#E9A6A6` / `#F2B5A8` | 渐变文字、图标高亮、轻提示 |
| 文字主色 | `#2E2C2A` | 主要标题、正文 |
| 文字辅助 | `#9B6E6E` | 标签、说明、占位符 |
| 边框/分割线 | `#FFE0D6` 至透明 | 极浅暖调边框 |

### 3.2 禁止色

以下颜色风格在内页中禁止作为大面积主视觉：

- 纯黑
- `#333333` 以下的深灰
- 暗紫色
- 高饱和冷蓝
- 偏脏的深青灰

### 3.3 字体系统

- 西文字体：`Inter`
- 备用：`SF Pro Text`、`Helvetica Neue`
- 中文字体：`PingFang SC`、`Microsoft YaHei`
- 字重：`300 / 400 / 500 / 600`

字号建议：

- 大标题：`24px / 28px`，`600`
- 模块标题：`20px`，`600`
- 正文：`15px`，`400`
- 辅助说明：`13px`，`400`
- 进度数字：`32px`，`800`

### 3.4 圆角与间距

- 卡片圆角：`32px`
- 按钮圆角：`60px`
- 输入框圆角：`40px`
- 页面左右内边距：桌面 `24px`，移动端 `16px`
- 卡片内边距：`24px`
- 栅格基础：`8px`
- 常用间距：`16px / 24px / 32px`

### 3.5 阴影与质感

- 卡片阴影：`0 12px 28px -8px rgba(199,84,84,0.12)`
- 按钮阴影：`0 4px 8px rgba(211,105,105,0.2)`
- 毛玻璃：仅导航栏、轻提示层可使用
- 取消任何黑色厚阴影、脏灰投影、大面积压屏阴影

一句话原则：

**阴影要像暖光，不要像雾霾。**

## 四、背景与构图规范

### 4.1 正确的背景构图

内页主背景必须采用：

- 浅奶油到底
- 淡粉至暖白渐变
- 局部柔光晕染
- 少量星点、花瓣、音符等超低透明度装饰

推荐构图：

1. 大面积浅色底
2. 顶部或侧边一处暖粉光晕
3. 卡片以白底悬浮其上
4. 少量局部插图或浅色情绪图卡

### 4.2 错误的背景构图

以下构图直接禁止：

- 深色现场照片整页平铺
- 上半屏摄影图、下半屏浅卡的强对撞结构
- 演唱会照片压在表单后面形成“舞台幕布感”
- 高对比聚光灯、暗部人群、彩带夜景成为主底

### 4.3 照片使用原则

如果确实要用演唱会、球赛现场图，只允许这样使用：

- 放入顶部小型轮播卡片
- 放入某个模块的情绪封面
- 放入空状态插画区域
- 经过去暗处理后作为浅色遮罩底图

照片处理要求：

- 提亮
- 降对比
- 增加白色蒙层
- 裁切掉压迫感强的人群暗部
- 保证文字可读性优先

## 五、页面结构与组件规范

## 5.1 顶部导航栏

### 目标

像一张温柔的品牌抬头，而不是系统菜单条。

### 结构

左侧：品牌区

- 主标题：`一站式赴约小管家`
- 标题采用渐变色：`#D96C6C → #F2B5A8`
- 搭配爱心图标或柔和圆形品牌徽记

副标题：

`把每一场期待，提前安排成更安心也更浪漫的赴约。`

右侧：导航链接

- 首页
- 创建计划
- 我的安排

### 样式

- 背景：半透暖白胶囊
- 内边距：`8px 20px`
- 激活态：文字稍加粗，下方细小红色圆点指示器
- 可有轻毛玻璃，但不允许发灰发脏

---

## 5.2 进度与心情卡片

### 位置

位于导航栏下方，作为本页第一块情绪型信息卡。

### 内容

左侧：

- 标题：`准备这次赴约`
- 小字：`FAN DATE PLAN · 追星助手版`

右侧：

- 圆形进度指示器
- 数字示例：`28%`
- 说明：`进度 · 安心筹备`

### 样式

- 背景：`纯白 → 浅杏色` 微渐变
- 无重边框
- 数字颜色：品牌主色
- 整体更像暖心状态卡，而不是项目看板

---

## 5.3 暖心引言区

### 目标

打掉系统感，建立“被陪伴”的心理预期。

### 样式

- 背景：`#FFF6F0`
- 左侧加粗边框：`6px #FFB7B2`
- 图标：对话气泡、星星、爱心、音符均可

### 文案语气

沿用陪伴式、选择题风格表达，例如：

`这次的小方向 —— 能选的我尽量都做成了选择题，少一点填表感，多一点被陪着往前走的感觉。`

要求：

- 不要像系统提示
- 不要写命令式句子
- 要像小管家在轻轻提醒你

---

## 5.4 步骤指示器

### 展示方式

`1/6 → 6/6` 胶囊式按钮横向排列。

### 样式

未完成：

- 背景：`#FFF4EF`
- 文字：`#BBA2A2`

当前步骤：

- 背景：`#FFDFD3`
- 文字：`#CB6E6E`
- 轻微内阴影

交互：

- hover / touch 时轻微上浮
- 不做强烈跳动动画

---

## 5.5 双栏表单区

### 总原则

这是整页最关键的部分。

必须把“填表系统”改造成“选择式赴约安排器”。

要求：

- 优先标签选择
- 次优下拉选择
- 最后才是手动输入
- 系统已有场馆优先直接选，不要默认让用户自己写

### 左侧卡片：定场馆和日期

标题：

`先定场馆和日期`

搭配地图针图标

字段建议：

- 活动类型：标签组  
  `演唱会 / 音乐节 / 见面会`
- 活动名称：短输入框
- 城市：下拉选择器
- 场馆：优先联动选择器
- 日期：日期选择
- 时间：时间选择

辅助文案：

`先把活动、城市和场馆选清楚，后面规则和当天节奏才会更准。`

### 右侧卡片：你最在意什么

标题：

`你最在意什么`

搭配星星图标

字段建议：

- 多选标签组：
  - 应援提醒
  - 出片安排
  - 物料交换
  - 散场返程
  - 场馆规则
- 出行方式标签：
  - 地铁
  - 自驾
  - 网约车
  - 粉丝包车
- 出发地：短输入框或下拉
- 票档：下拉或分段选择

辅助文案：

`把应援、出片、物料和返程说明白，这份手册才会更像专门为你写的。`

---

## 5.6 表单组件规范

输入框与下拉框必须具备以下特征：

- 白底或浅奶油底
- 边框极浅暖灰粉
- 聚焦时出现柔和粉色描边
- 占位符使用玫瑰棕色辅助文案
- 高度统一，不要忽大忽小
- 不允许出现深灰底输入框

标签按钮必须：

- 轻、圆、软
- 选中态有浅暖色高亮
- 点击反馈立即可见
- 文字不能太黑太硬

---

## 5.7 浪漫操作按钮

主按钮文案：

`生成我的浪漫赴约手册 ✨`

样式：

- 渐变：`#FFB7AF → #FFAAA0`
- 白色文字
- 圆角：`60px`
- 按钮阴影：暖色柔影

交互：

- hover / 点击时轻微缩放
- 阴影稍加强
- 不允许出现生硬的矩形主按钮

---

## 5.8 轻提示与装饰条

在主表单区下方增加两类轻提示卡：

- 出片安排
- 应援提醒

样式：

- 半透暖色背景
- 圆角 `28px`
- 无重边框
- 信息像贴心补充，不像警告框

底部 footer：

- 爱心图标
- 温柔标语
- 文字 `12px`
- 颜色 `#DBB6B6`

## 六、交互与动效规范

| 交互行为 | 动效说明 |
| --- | --- |
| 标签点击 | `0.96 → 1` 的轻缩放，背景变为 `#FFE3DA` |
| 按钮 hover/点击 | `0.2s` 缩放 + 阴影过渡 |
| 卡片 hover | 上移 `2px`，阴影略增强 |
| 生成手册反馈 | 底部浮出 toast，`2.5s` 后淡出 |
| 页面加载 | 轻微淡入 |

生成反馈文案建议：

`✨ 你的专属赴约手册正在生成 ✨`

明确禁止：

- 闪烁
- 剧烈弹跳
- 赛博风高频动效
- 长时间加载动画
- 带压迫感的暗色浮层

---

## 七、响应式与适配

### 桌面端（≥1024px）

- 双栏并排
- 左右留白充足
- 最大内容宽度：`1280px`

### 平板端（768px - 1023px）

- 适当缩小卡片内边距
- 双栏可继续并排
- 比例调整为 `1:1`

### 移动端（≤767px）

- 双栏改为单列
- 导航栏换行或切为后续菜单形态
- 卡片圆角改为 `24px`
- 按钮圆角改为 `50px`
- 标题缩小为 `22px`
- 正文缩小为 `14px`

核心交互要求：

- 单手握持时，主要操作区落在拇指热区
- 标签和按钮必须足够大，避免点选困难

## 八、情感化设计细节

### 8.1 图标风格

- 使用线性或柔和实心图标
- 避免尖锐科技感图标
- 避免高冷工具风 icon

推荐元素：

- 爱心
- 星星
- 云朵
- 音符
- 日历
- 地图针

### 8.2 文案语气

统一采用：

- 第二人称“你”
- 第一人称“我 / 小管家”

语气要求：

- 轻提醒
- 被理解
- 温柔但不幼态
- 专业但不系统

### 8.3 惊喜元素

页面可加入：

- 右上角浅浅的星星
- 底部花瓣或音符浅印
- 透明度低于 `0.05`

目的：

- 提升浪漫气氛
- 不干扰信息阅读

### 8.4 完成态反馈

当必选项完成后：

- 进度数字自动增加
- 可增加极轻微微光效
- 这属于加分项，不做强制复杂动画

## 九、针对创建页的专项要求

创建页是用户最容易产生“填表烦躁感”的地方，因此需要额外强调：

### 9.1 交互原则

- 能选就别填
- 系统已有场馆必须优先联动选择
- 城市选定后，只显示对应城市的主流场馆
- 活动类型、出行方式、在意事项尽量用标签
- 票档优先用常见选项，不让用户从零输入

### 9.2 结构原则

创建页应该像：

- 被引导安排一次赴约

而不是：

- 在填一份复杂表单

### 9.3 视觉原则

创建页不允许再出现：

- 深色现场大图当底
- 明暗反差极强的摄影图铺底
- 一大坨发灰发暗的背景把浅卡片包住

创建页推荐构图：

- 顶部浅色欢迎区
- 中部双栏选择区
- 下方温柔提示卡
- 局部小图或插画点缀

---

## 十、针对赴约手册页的专项要求

赴约手册页应更像一本随身小手册，而不是 AI 报告页。

### 10.1 视觉气质

- 明亮
- 柔和
- 可收藏
- 像认真整理过的赴约页签

### 10.2 信息顺序

优先级建议：

1. 这次赴约的封面信息
2. 你最在意的提醒
3. 场馆规则与禁带信息
4. 当天节奏 / 时间线
5. 出片 / 应援 / 物料 / 会合
6. 返程安排

### 10.3 禁止风格

- 禁止做成深色仪表盘
- 禁止做成“战报”“作战页”“控制台”
- 禁止冷冰冰的 AI 结果清单

---

## 十一、资源清单

- 字体：Google Fonts `Inter`，备用系统字体
- 图标：Font Awesome 6 Free 或同类柔和风图标集
- 背景纹理：极浅噪点，透明度 `< 0.02`
- 装饰插画：手绘风星星、云朵、花瓣、音符

---

## 十二、设计验收检查点

内页最终验收必须满足以下条件：

- 没有任何背景色深于 `#E8D8D0`
- 不存在整页深色照片背景
- 所有主要卡片都落在明亮浅底上
- 文字与背景对比度 `≥ 4.5:1`
- 选择型交互占表单字段 `60%` 以上
- 页面中至少存在 `2` 处情感化文案
- 主按钮文案不使用“提交”“确定”
- 移动端主要操作区落在拇指热区
- 页面整体感受必须是“明亮、漂亮、轻盈、被陪伴”

### 最终判断标准

如果用户第一眼仍然会说：

- “阴暗”
- “像后台”
- “还是很重”
- “不像追星赴约产品”

那就视为设计不合格，需要重做。

---

## 十三、执行顺序建议

按这份文档落地时，建议顺序如下：

1. 先重做创建页背景和整体构图
2. 再统一卡片、标签、输入框、按钮样式
3. 再改进度卡、暖心引言区、步骤指示器
4. 再同步到赴约手册页、我的安排页、分享页
5. 最后补局部照片、插画、动效点缀

原因很简单：

先救整体气质，再补精致细节。
