# 一站式赴约小管家 · 内页 UI 设计文档

版本：V3.0 呼吸感 × 科技感 × 浪漫感平衡版  
文档状态：定稿  
适用范围：创建页、赴约手册页、我的安排页、分享页  
本版目标：解决“太粉嫩、太单调、没层次、没高级感、没科技感”的问题，把内页从“浅粉表单页”升级成“有结构、有呼吸感、有微妙科技感的追星赴约产品”。

---

## 一、为什么要重做

上一版的问题不在于“不够亮”，而在于亮得太单一：

- 颜色过度依赖粉色，导致页面气质偏甜、偏平，缺少层次
- 明亮有了，但空间组织不高级，页面像一整片浅粉卡片
- 浪漫感被误解成“堆粉色”，缺少更成熟的材质、光感和节奏
- 科技感几乎没有，页面不够像 AI 产品
- 信息架构还不够“复杂但有序”，缺少成熟旅游平台那种结构感

这次的方向不是回到暗色，而是：

**保留明亮温柔的基调，但加入更克制的高级配色、更强的信息层级、更轻的玻璃层叠感和更成熟的页面构图。**

---

## 二、参考方向总结

本版参考了几类成熟产品与页面思路，但不照抄视觉表皮。

### 2.1 Explora Journeys

可借鉴的核心不是“豪华邮轮”，而是：

- 留白充足
- 构图开阔
- 用克制色彩建立高级感
- 用大图/大段空间承接情绪，但不压信息
- 信息多时仍然有呼吸感

我们借鉴它的：

- 开阔版面
- 高级中性色
- 分区节奏
- 情绪型顶部封面区

不借鉴它的：

- 过于奢侈品化
- 过重的品牌距离感

### 2.2 Velaris 风格描述

你提到的“层叠幻境、科技感、玻璃拟态、粒子系统”非常有价值。  
我没有找到一个足够稳定、明确可核实的同名目标站点，因此这里把它视为一种你认可的风格方向，而不是严格复刻某一个具体页面。

可借鉴的核心：

- 多层卡片叠放
- 轻玻璃感而不是重玻璃感
- 背景里有柔和光晕和细颗粒
- 不是赛博朋克，而是“科技诗意”

### 2.3 Wonder Planner / Layla / AI 旅行产品

这些产品给我们的启发更直接：

- AI 产品首页和表单页要像“我来帮你开始”，不是“请先填写”
- 情绪型文案和功能型流程要同时成立
- 复杂信息应拆成“先给你第一步，再逐渐展开”
- 可信度来自清晰结构，不只是漂亮界面

### 2.4 GoBreck

它最值得借的不是色彩，而是结构：

- 信息很多，但组织清楚
- 面向真实出行场景
- 强调快速进入关键信息
- 默认摘要，必要时再展开

这个对“赴约手册页”尤其重要。

## 三、本版新的视觉方向

### 3.1 核心关键词

- 明亮，但不发飘
- 浪漫，但不幼态
- 科技，但不冰冷
- 精致，但不奢靡
- 复杂，但不杂乱

### 3.2 最终气质定位

这不是：

- 旅游网站换皮
- 粉色表单工具
- 深色科技后台

而是：

**一个面向演唱会、音乐节、球赛赴约的 AI 小管家产品。**

它应该让用户感受到：

- 我被理解
- 我被陪着安排
- 页面很漂亮，但不是空好看
- 这是一个成熟产品，不是试验稿

---

## 四、全新配色系统

本版不再以“粉色铺满”为主，而改成：

**奶油白 + 沙色 + 雾玫瑰 + 香槟金 + 海军蓝少量压轴 + 雾青绿点缀**

### 4.1 颜色角色

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 主背景 | `#FCF8F4` | 页面主底，温暖但不粉腻 |
| 背景渐变辅色 | `#F9EFEA` / `#F7F2EE` | 渐变过渡、顶部气氛区 |
| 主卡片 | `#FFFFFF` | 主要内容承载 |
| 次级卡片 | `#FBF6F2` | 辅助区块、摘要卡 |
| 品牌强调色 | `#C45C66` | 关键 CTA、重点文案 |
| 浪漫辅助色 | `#E6A8A1` | 选中标签、柔和高亮 |
| 香槟金 | `#D9B98E` | 高级感点缀、分割、数字 |
| 深海军蓝 | `#273248` | 用于少量压标题、科技层次、深色信息点 |
| 雾青绿 | `#9EBDB4` | 科技感平衡色、状态提示、轻微图层点缀 |
| 主文字 | `#2F2B2A` | 标题与正文 |
| 次文字 | `#826D69` | 辅助说明 |
| 弱边框 | `#EEDFD5` | 边框和分隔 |

### 4.2 新原则

- 粉色只做情绪点缀，不做全页统治色
- 深海军蓝只做小面积结构压舱，不回到暗黑风
- 香槟金和雾青绿负责把“浪漫”拉回“精致”

### 4.3 禁止项

- 禁止整页粉
- 禁止整页深蓝
- 禁止紫粉霓虹滥用
- 禁止高饱和糖果色堆叠
- 禁止所有模块都长得一样

## 五、材质与层次语言

### 5.1 三层系统

页面必须具备明确三层，不再是一整片平铺卡片：

1. **底层 Atmosphere**
   - 奶油白渐变底
   - 柔和光晕
   - 超浅颗粒或噪点
   - 极轻漂浮元素

2. **中层 Structure**
   - 白色/浅米色大卡片
   - 明确栅格
   - 模块间留白
   - 结构型分区

3. **上层 Accent**
   - 浅玻璃感小卡
   - 标签、数字、状态胶囊
   - 少量高亮按钮
   - 轻动效反馈

### 5.2 玻璃感使用原则

玻璃感只可用于：

- 顶部导航
- 情绪摘要小卡
- 顶部封面信息层
- 轻提示层

不允许：

- 全页面都玻璃
- 所有卡片都毛玻璃
- 毛玻璃把文字搞糊

一句话：

**玻璃感是“层次调味料”，不是主食。**

## 六、背景与构图

### 6.1 新的背景构图原则

主背景不再是“粉底 + 卡片”，而是：

- 浅奶油底
- 淡沙色与淡玫瑰晕染
- 局部加入低透明雾青或海军蓝灰气氛层
- 整体看起来像晨光，不像糖霜

### 6.2 禁止使用的背景方式

- 禁止整页深色现场摄影图
- 禁止上半屏暗图压下半屏浅表单
- 禁止大面积彩带夜景图作为表单底
- 禁止“演唱会现场照片 + 白卡硬盖”这种突兀拼接

### 6.3 如果要用图片

图片只能用于：

- 顶部横向情绪轮播
- 模块局部封面图
- 分享页局部气氛图

图片必须经过：

- 提亮
- 去暗
- 降对比
- 柔焦
- 蒙一层暖白或浅米遮罩

目标：

**画面是“有故事的空气感”，不是“夜场海报”。**

---

## 七、信息架构原则

### 7.1 参考 GoBreck 的“结构优先”

我们的页面不能只会好看，必须会组织复杂信息。

适用于“赴约手册页”的原则：

- 默认先给摘要
- 重要信息前置
- 细节可展开
- 长内容拆成多模块
- 一个模块一个重点，不要堆满

### 7.2 页面节奏原则

每页都应该有清楚的阅读节奏：

1. 情绪封面
2. 当前关键摘要
3. 主操作或主信息区
4. 次级说明区
5. 补充信息区

用户不应该一上来就被大量字段和信息轰炸。

## 八、创建页专项设计

### 8.1 气质

创建页不是表单页，而是“开始安排这次赴约”的引导页。

### 8.2 结构

建议分成 4 层：

1. **顶部封面区**
   - 大标题
   - 一句陪伴式引导
   - 小型情绪图或气氛渐变

2. **左侧进度/摘要区**
   - 当前完成度
   - 已选关键信息
   - 轻提示

3. **主表单区**
   - 双栏
   - 选择题优先
   - 标签、下拉、联动选择器优先

4. **场馆规则预览区**
   - 默认摘要卡
   - 命中后展开更多

### 8.3 交互原则

- 城市先选
- 场馆联动展示
- 能选就别填
- 活动类型、应援重点、出行方式尽量标签化
- 高级选项折叠，默认不干扰主流程

### 8.4 视觉重点

按钮和关键状态不要只靠粉色突出，应同时依靠：

- 留白
- 尺寸
- 位置
- 字重
- 轻阴影

## 九、赴约手册页专项设计

### 9.1 定位

这是“随身赴约手册”，不是“AI 报告结果页”。

### 9.2 构图

顶部封面建议：

- 左侧：活动名、场馆、日期、你最在意的事
- 右侧：小型玻璃摘要卡
- 背景：浅色气氛层，不要暗图

### 9.3 模块层级

建议顺序：

1. 封面摘要
2. 赴约安心度
3. 场馆规则
4. 时间线
5. 防踩坑摘要
6. 穿搭 / 出片 / 应援 / 会合
7. 返程安排

### 9.4 模块形态

不同模块不能都长一个样：

- 分数模块：更偏数字主视觉
- 规则模块：更偏摘要卡 + 标签
- 时间线模块：更偏纵向节奏
- 提醒模块：更偏贴纸卡 / 小块卡片
- 返程模块：更偏实用指令卡

## 十、我的安排页专项设计

### 10.1 方向

不要像列表库，要像“已经整理好的赴约收藏册”。

### 10.2 设计原则

- 卡片有主次尺寸变化
- 支持筛选，但不过度工具化
- 每张卡必须有“这次赴约的记忆点”
- 保留轻情绪感，而不只是数据

### 10.3 卡片内容建议

- 活动名
- 场景
- 城市/场馆
- 日期
- 这次主目标
- 快速打开 / 再次生成 / 分享

---

## 十一、分享页专项设计

### 11.1 目标

朋友打开时，第一眼要觉得：

- 这东西好精致
- 这份安排很完整
- 我也想做一份

### 11.2 结构

- 顶部简洁分享引导
- 中间直接展示手册主体
- 下方 CTA 要明显但不打扰

### 11.3 风格

比创建页更简洁，比手册页更轻一点。

---

## 十二、动效与科技感

科技感不是靠暗色，而是靠“空间反馈”。

### 12.1 建议使用

- 卡片 hover 轻上浮
- 毛玻璃层轻微 blur
- 按钮阴影缓动
- 页面 section 淡入
- 标签点击有细微压缩反馈
- 轮播区有缓慢切换
- 极轻颗粒与漂浮元素

### 12.2 禁止使用

- 高速粒子特效
- 赛博光污染
- 强闪烁
- 大面积玻璃反射
- 所有元素都飘

一句话：

**科技感来自秩序、响应和层叠，不来自发光乱闪。**

---

## 十三、文案语气

继续保留陪伴感，但要更成熟：

- 少一点“可爱”
- 多一点“被理解”
- 少一点系统口令
- 多一点轻柔提示

推荐语气：

- “先把这几个关键点定下来，后面就会顺很多。”
- “我先替你把容易慌的地方拎出来。”
- “这一步不用想太复杂，先选最接近你的就好。”

---

## 十四、设计验收标准

本版验收必须同时满足：

### 14.1 色彩

- 不能再是一整页粉
- 页面至少有 3 类主辅色协同工作
- 粉色占比必须降到“点缀级”，不是“统治级”

### 14.2 层次

- 用户第一眼能区分背景层、结构层、强调层
- 不同模块有不同表现方式
- 页面不再像一堆同样的浅色卡片

### 14.3 气质

- 明亮
- 高级
- 温柔
- 有一点科技感
- 有一点浪漫感
- 但不幼态

### 14.4 交互

- 选择型交互 ≥ 60%
- 场馆优先联动选择
- 重要信息默认摘要化
- CTA 明显但不突兀

### 14.5 否决条件

如果用户仍然会第一反应说：

- “太粉了”
- “太单调了”
- “没层次”
- “像半成品”
- “不像成熟产品”

则视为本轮设计失败。

---

## 十五、执行顺序

这次建议严格按顺序落地：

1. 先重做内页全局背景和配色系统
2. 再重做创建页构图
3. 再重做赴约手册页模块层级
4. 再统一我的安排页和分享页
5. 最后补玻璃层、细颗粒、轮播和轻动效

原因：

先救空间层次，再补浪漫和科技细节。
