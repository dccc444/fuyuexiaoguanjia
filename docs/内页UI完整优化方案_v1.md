# 赴约小管家 · 全量内页 UI 完整优化方案 V1

状态：执行前总方案  
适用范围：创建计划页、赴约手册页、我的安排页、分享页、后续专区页  
方法：综合 `taste-skill`、`impeccable`、`ui-ux-pro-max` 的审美/层级/交互思路，并结合旅游攻略、AI 行程规划产品参考做统一规划。

---

## 1. 这份方案解决什么问题

当前内页主要有 4 个问题：

1. 主题反复横跳  
   一会儿浅底、一会儿深底，首页和内页像两个产品。

2. 可读性差  
   特别是正文、小字、占位符、来源说明，颜色偏灰偏虚。

3. 缺乏成熟产品的秩序感  
   卡片多但层次不稳，模块之间像堆出来的，不像经过信息架构梳理。

4. 功能没有围绕 `功能模块.docx` 收束  
   “衣、食、住、行、票、社交” 应该是内页骨架，而不是零散插入。

这份方案的目标不是“再调一版颜色”，而是：

`把所有内页统一成一个完整系统，让它像成熟的追星赴约产品。`

## 3. 统一设计结论

### 3.1 总体定位

产品应该是：

`旅行规划产品的秩序感 + 追星助手的情绪感 + AI 助手的陪伴感`

不能是：

- 旅游 OTA 复制品
- 暗色后台面板
- 粉色堆砌页
- 只会做淡色大白板

### 3.2 内页世界观

首页是品牌入口，内页是“真正开始安排这次赴约”的工作区。

所以内页需要：

- 比首页更清楚
- 比首页信息密度更高
- 但仍然属于首页同一家族

结论：

- 内页保留浅底
- 用更稳定的中性色和更强的层级解决清晰度问题
- 允许少量冷色和图片引入层次

## 4. 视觉系统

### 4.1 画布结构

- 主背景：浅暖白，不再改成整页深色
- 内容卡：白色或极浅奶油白
- 局部强化：暖粉、杏橙、雾蓝、薄荷绿
- 图片：只做局部引导，不做整页铺底

### 4.2 色彩策略

不再单一粉色。

建议采用 4 组协同色：

1. 品牌暖色  
   `#F08A93` / `#D96575`

2. 浅奶油中性底  
   `#FFF8F6` / `#FFFDFC` / `#FFF4EF`

3. 清爽冷色辅助  
   `#DCEBFF` / `#7FAEEA`

4. 柔和自然辅助  
   `#DDF3EA` / `#73BFA2`

### 4.3 中性色阶

这是所有内页必须统一的基线：

- H1/H2：`#302A36`
- 主正文：`#4A4455`
- 次正文：`#6A6476`
- 辅助字：`#877F93`
- 占位符：`#9D96A8`

硬规则：

- 正文不能再用接近背景的灰粉色
- 模块说明文字必须一眼可读

### 4.4 层级术语落地

后续评审统一使用这些词：

- `Elevation`
- `Negative Space`
- `Grid Layout`
- `Card Nesting`
- `Divider Hierarchy`
- `Type Scale`
- `Neutral Scale`
- `Functional Color`

## 5. 动效系统

这次动效以“精致反馈”为主，不靠重炫技。

### 5.1 允许动效

- `Fade In`
- `Soft Lift`
- `Pressed Offset`
- `Focus Highlight`
- `Step Form Wizard`
- `Chip Filter`
- `Mount Transition`
- `Sticky Pin`

### 5.2 适合首页但不适合内页的动效

- 开屏遮罩
- 全局探照灯
- 大幅视差
- 复杂粒子系统

内页更适合：

- 表单聚焦高亮
- 标签选择反馈
- 悬浮小幅抬升
- 切换时轻渐入

## 6. 全量内页信息架构

这次按 `功能模块.docx` 重组，不再按“页面长什么样”想，而是按用户一次赴约真实路径来组织。

### 6.1 创建计划页

作用：

- 收集这次赴约的基础条件和追星偏好

模块顺序：

1. 基础信息  
   活动类型、活动名称、艺人/主队、城市、场馆、日期、时间

2. 出行条件  
   出发地、同行人数、预算、出行方式

3. 追星重点  
   应援、出片、物料、搭子、返程、场馆规则

4. 深度偏好  
   衣、食、住、行、票、社交

5. 场馆规则预览  
   根据场馆即时展示

UI 原则：

- 能选就别填
- 用 `Chip Filter`、`Auto Complete Dropdown`、`Step Form Wizard`
- 场馆和城市必须联动

### 6.2 赴约手册页

作用：

- 生成后的主结果页

结构：

1. 顶部总览  
   活动、艺人、城市、场馆、时间、赴约安心度

2. 六大摘要区  
   衣 / 食 / 住 / 行 / 票 / 社交

3. 规则与提醒区  
   场馆规则、重点风险、入场清单

4. 当天节奏区  
   时间线、会合、返程、散场

5. 底部行动区  
   复制分享、再次生成、回到我的安排

### 6.3 我的安排页

作用：

- 管理历史赴约

结构：

1. 顶部筛选  
   城市、场景、时间、状态

2. 卡片列表  
   摘要信息、场馆、日期、艺人、进度

3. 快捷动作  
   查看手册、复制计划、再次生成、分享

### 6.4 分享页

作用：

- 轻量展示给别人看的摘要页

结构：

1. 顶部分享横幅
2. 关键信息总览
3. 六大模块精选摘要
4. CTA：我也创建一份

## 7. 按功能模块落到页面

### 7.1 衣

放在：

- 创建页：偏好选择
- 手册页：穿搭与妆造建议卡

推荐组件：

- `Chip Filter`
- `Support Card`
- `Checklist Summary`

### 7.2 食

放在：

- 创建页：饮食偏好 / 补给重点
- 手册页：场馆前、场馆中、散场后三段式补给建议

推荐组件：

- 时间段分组卡
- 推荐清单

### 7.3 住

放在：

- 创建页：是否跨城、是否需要住宿
- 手册页：住哪里最顺、备选策略

推荐组件：

- 三类住宿策略卡
- 位置优先级摘要

### 7.4 行

放在：

- 创建页：出发地、出行方式、返程压力
- 手册页：到场、进场、散场、返程四段路线

推荐组件：

- 时间线
- 风险 pill
- 交通建议卡

### 7.5 票

放在：

- 创建页：票档 / 区域 / 分区
- 手册页：看台/内场/票档差异提醒

推荐组件：

- 分区摘要卡
- 重点提示卡

### 7.6 社交

放在：

- 创建页：搭子、会合、物料计划
- 手册页：会合提醒、物料提醒、一个人去也不慌

推荐组件：

- 协作卡
- 会合提醒卡
- 社交清单

## 8. 页面逐一优化方案

## 8.1 创建页优化方案

当前问题：

- 浅底方向是对的，但说明字还虚
- 选择器不够像成熟旅行产品
- 结构还偏“表单”

优化目标：

- 像 AI 规划工作台，而不是录入页

具体改法：

1. 顶部 Hero 压缩  
   不再过高，不重复叙事

2. 左侧进度变轻  
   只做导航和完成度，不抢主表单

3. 主操作区像搜索面板  
   借 Trip.com / 携程的清晰感

4. 城市和场馆增强  
   热门城市 chip、联动场馆卡、搜索补充

5. 追星偏好改成可扫读  
   选择型卡片，不要一堆小输入框

## 8.2 手册页优化方案

当前问题：

- 曾被错误切到暗色版本
- 浅底方案下文字不够强
- 六大模块还不够像成体系的攻略卡

优化目标：

- 像一份成熟的赴约攻略手册

具体改法：

1. 浅底保留
2. 概览区像 itinerary summary
3. 六大模块形成稳定色系节奏
4. 风险、清单、规则做更强功能色区分
5. 时间线保持清楚，不要过装饰

## 8.3 我的安排页优化方案

当前问题：

- 还是偏普通列表

优化目标：

- 像旅行产品里的“我的行程”

具体改法：

1. 增加筛选条
2. 卡片摘要统一
3. 快捷动作常驻
4. 状态和最近活动有视觉优先级

## 8.4 分享页优化方案

当前问题：

- 太像详情页复制版

优化目标：

- 一屏能懂这次赴约安排

具体改法：

1. 分享横幅更轻
2. 摘要信息更紧凑
3. 只展示最值钱的模块
4. 强 CTA 保留

## 9. 执行顺序

### 第一阶段：视觉回正

1. 手册页恢复浅底
2. 分享页恢复浅底
3. 全内页统一文字对比度

### 第二阶段：创建页成型

1. 城市 / 场馆联动选择
2. 追星偏好选择型表单
3. 表单工作台化

### 第三阶段：手册页按六大模块重构

1. 衣
2. 食
3. 住
4. 行
5. 票
6. 社交

### 第四阶段：我的安排 / 分享页统一

1. 历史卡片
2. 快捷动作
3. 分享摘要

## 10. 验收标准

### 10.1 视觉

- 内页与首页属于同一个产品家族
- 不是满页粉，也不是整页暗黑
- 至少有一套稳定的中性色文字系统

### 10.2 可读性

- 正文不再发灰发虚
- 占位符和小字也能清楚看见
- 重点模块标题一眼能扫到

### 10.3 功能

- 创建页能快速完成
- 手册页一眼能看到衣食住行票社交
- 我的安排和分享页都能承担真实使用场景

### 10.4 产品感

- 更像“追星赴约助手”
- 不像普通旅游表单
- 不像后台系统

## 12. 最终结论

后续所有内页改版统一按这条路线走：

1. 先把浅底方案做对
2. 用文字层级和中性色阶解决可读性
3. 用 OTA/AI 行程规划产品的秩序感解决“乱”
4. 用 `衣 / 食 / 住 / 行 / 票 / 社交` 作为手册和创建页的主骨架

这份文档是后续所有内页优化的总基准。
