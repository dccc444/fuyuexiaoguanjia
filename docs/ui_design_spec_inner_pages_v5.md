# 一站式赴约小管家 · 内页 UI 设计文档 V5

状态：定稿前评审版  
目标：把内页从“方向飘忽”拉回到统一、清楚、浪漫且高级的方案。  
当前结论：内页不切深色主题，回到浅底版本；重点修正文字对比度、层次感、模块密度和参考体系。

## 1. 先修正什么

本轮第一优先级不是重做结构，而是回滚错误配色方向。

必须执行：

1. 内页配色回到你给的第二张图那种浅底版本。
2. 禁止继续沿用上一轮深灰蓝大底的手册页。
3. 不再把“提高高级感”误做成“整体变暗”。
4. 当前内页只修文字可读性、信息层级、组件精致度，不切换主题世界观。

一句话原则：

`保留浅底、提高清晰度、增加层次，不再切成暗色产品。`

---

## 2. 当前问题诊断

### 2.1 第一张图的问题

- 深色背景把产品拉成了另一个系统，和首页不属于同一家族。
- 模块颜色虽然更多，但整体发闷，失去“赴约”的轻盈感。
- 大面积深色降低了情绪价值，变成“工具后台感”。

### 2.2 第二张图的问题

- 配色方向更接近对的，但文字对比度不够。
- 正文、辅助文案、输入框占位符都过浅。
- 大面积浅粉和浅奶油缺少更明确的中性色阶支撑。
- 层次主要靠渐变，不是靠真正的信息层级。

### 2.3 真正该修的不是“深浅”，而是这 4 件事

1. `Type Contrast`
   标题、正文、辅助文字、占位文字没有拉开层级。
2. `Neutral Scale`
   缺一组稳定的中性色，导致所有字都像蒙了一层粉灰。
3. `Elevation Hierarchy`
   卡片、输入框、标签、次级模块的抬升关系不清楚。
4. `Content Density`
   内页需要更像成熟旅行规划产品，不能只是一页淡色块。

### 3.1 你提供的高价值参考

#### 携程酒店页

借鉴点：

## 4. 新的内页视觉方向

### 4.1 定位

不是：

- 粉色表单工具
- 暗色数据面板
- 旅游 OTA 复制品

而是：

`像成熟旅行规划产品一样清楚，像追星助手一样懂情绪。`

### 4.2 视觉关键词

- 明亮
- 清楚
- 有层次
- 轻浪漫
- 轻攻略感
- 轻 AI 感

### 4.3 总体设计语言

采用：

- `浅底主画布`
- `稳定中性色文字系统`
- `暖色品牌强调`
- `少量清爽冷色辅助`
- `更成熟的旅行攻略式卡片布局`

不再采用：

- 大片低对比粉色正文
- 过多雾化渐变导致的发灰
- 大面积深色反转主题

## 6. 可读性修正规范

### 6.1 标题层级

- H1：`#302A36`，字重 `700`
- H2：`#3B3445`，字重 `700`
- H3：`#4A4455`，字重 `600`

### 6.2 正文层级

- 主要正文：`#4A4455`
- 次要正文：`#6A6476`
- 注释说明：`#877F93`

### 6.3 输入区

- Label：`#5E5669`
- Placeholder：`#A19AAA`
- 输入值：`#362F40`
- 边框默认：`rgba(219, 207, 221, 0.88)`
- Focus 边框：`#F0A0A8`

### 6.4 卡片内文字

卡片里所有 `p` 文本不能再使用接近背景的灰粉色。

统一要求：

- 普通模块正文至少 `#5C5666`
- 攻略建议正文至少 `#4E4858`
- 来源和补充说明至少 `#7B7485`

## 7. 结构升级原则

### 7.1 创建页

参考 Wonderplan + 马蜂窝 + 你现有首页。

保留：

- 左侧进度
- 中间主表单
- 分步骤节奏

升级：

1. 城市、场馆、活动类型全部优先做选择器
2. 输入区密度更像成熟旅行搜索面板
3. 热门城市、主流场馆做成可点选 chip + 列表卡
4. 表单块像“整理赴约计划”，不是“系统录入”

### 7.2 赴约手册页

参考 Tripadvisor / Trip.com 内容卡片秩序。

结构应该变成：

1. 顶部概览区
   活动、城市、场馆、日期、艺人、安心度
2. 重点摘要区
   衣 / 食 / 住 / 行 / 票 / 社交
3. 规则与提醒区
   场馆规则、风险提示、入场清单
4. 当天节奏区
   时间线、会合、返程

重点：

- 用浅底卡 + 清晰分区
- 不用大块发灰渐变去装饰

### 7.3 我的安排页

参考攻略聚合页，不做白板式列表。

应该有：

- 时间排序
- 场景筛选
- 卡片摘要
- 再次生成
- 复制计划

### 7.4 分享页

分享页不是系统详情页副本。

应该更像：

- 一张可转发的赴约摘要海报
- 一段重点信息
- 三到四张核心模块摘要

## 8. 动效规范

这次不追求复杂炫技，重点做精致反馈。

### 8.1 允许使用

- `Fade In`
- `Soft Lift`
- `Pressed Offset`
- `Focus Highlight`
- `Chip Filter`
- `Mount Transition`
- `Sticky Pin`

### 8.2 暂不使用

- 大范围粒子系统
- 持续探照灯跟随
- 复杂 3D 视差
- 高强度 spring 动效

原因：

- 当前先解决结构和可读性
- 避免页面在未稳定前再显得“花但不好用”

## 9. 按功能模块的页面规划

基于 `功能模块.docx`，内页后续按这 6 类落位。

### 9.1 衣

- 穿搭建议
- 妆造提醒
- 出片友好提醒

UI 形式：

- 轻攻略卡
- 标签筛选
- 一眼可扫读的短建议块

### 9.2 食

- 场馆周边补给
- 排队前后吃什么
- 散场夜宵建议

UI 形式：

- 推荐卡片

### 9.3 住

- 住哪里最顺
- 离场馆近 / 返程方便 / 性价比

UI 形式：

- 三象限推荐卡
- 酒店策略摘要卡

### 9.4 行

- 到场方式
- 入场动线
- 散场返程
- 地铁 / 打车 / 高铁 / 飞机提醒

UI 形式：

- 时间线 + 行动建议卡

### 9.5 票

- 票档
- 看台分区
- 视角提醒
- 入场时间

UI 形式：

- 分区信息卡
- 重点提示卡

### 9.6 社交

- 搭子
- 会合
- 物料
- 周边交换

UI 形式：

- 协作感卡片
- 会合点摘要
- 提醒清单

## 11. 设计验收标准

### 11.1 配色

- 内页必须保持浅底
- 不再出现整块深蓝灰主背景
- 页面不是“满页粉”，必须有中性色和少量冷色平衡

### 11.2 可读性

- 正文和背景对比必须明显提升
- 用户截图不再出现“字看不清”
- 标签、辅助字、来源字也不能发白发灰

### 11.3 结构

- 创建页像成熟规划工具
- 手册页像一份攻略式赴约手册
- 模块主次清楚，不乱堆卡片

### 11.4 情绪

- 保留浪漫和温柔
- 但高级感来自秩序与层次，不来自堆粉和乱渐变

## 13. 最终结论

这轮设计方向不再争论：

1. 内页回到第二张图这种浅底方案
2. 只提升文字清晰度和层次感
3. 不再切深色主题
4. 视觉上吸收 OTA 与 AI 行程规划产品的秩序感
5. 功能上严格围绕 `衣 / 食 / 住 / 行 / 票 / 社交`

后续代码改动必须以本文件为唯一基准。
