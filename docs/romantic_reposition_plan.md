# 一站式赴约小管家：产品概念重规划

## 1. 先说结论

“赴约作战书”这个表达不适合当前产品。

原因不是它不清楚，而是它把产品的情绪带偏了。  
`作战书` 更像是：

- 高压
- 防御
- 生硬
- 功能理性
- 像在避免翻车，而不是期待一场见面

但“赴约”本身天然带着这些情绪：

- 期待
- 仪式感
- 心动
- 认真准备
- 为一场见面腾出时间和心情

所以这个产品不应该被包装成“应对风险的战术工具”，而应该被包装成：

`一份陪你把这次赴约安排得更顺、更美、更值得期待的行程灵感与贴心提醒`

也就是说：

- 底层能力仍然保留规划、提醒、规则、清单、路线
- 但外层表达要从“防翻车”改成“帮你安心赴约、认真赴约、浪漫赴约”

## 2. 新的产品语言方向

## 2.1 不再使用“作战书”

建议整体下线：

- 作战书
- 成功率
- 防踩坑雷达
- 指挥中心

这些词都偏硬、偏任务、偏工具，不适合你这个产品的气质。

## 2.2 新的核心表达建议

建议把产品核心产物改成以下几类表达之一。

### 方案 A：赴约计划

最稳，通用性最强，也最容易被用户理解。

适合表达：

- 帮你把这场活动安排清楚
- 不冒犯、不装、不生硬
- 适合演唱会 / 音乐节 / 球赛共用

可对应页面文案：

- 创建赴约计划
- 我的赴约计划
- 分享这份赴约计划

### 方案 B：赴约手册

比“计划”更有内容感，也比“作战书”柔和很多。

适合表达：

- 有点仪式感
- 有点收藏感
- 像一份为这次活动准备好的贴身指南

可对应页面文案：

- 生成我的赴约手册
- 查看赴约手册
- 分享这份赴约手册

### 方案 C：赴约小抄

更轻、更活泼、更社交化，适合年轻用户，但不够通用。

适合表达：

- 不想太正式
- 更像“朋友替你整理好重点”

可对应页面文案：

- 一键生成赴约小抄
- 今天照着小抄走

### 方案 D：赴约陪伴卡 / 赴约卡片

最浪漫，视觉化空间最大，但更适合做分享页或活动当天页，不适合做整个产品主命名。

适合表达：

- 分享感
- 精致感
- 更偏“内容卡片产品”

## 2.3 推荐选择

我建议采用：

`产品主概念 = 赴约计划`
`内容产物名称 = 赴约手册`

这样层级最顺：

- 用户先创建一份 `赴约计划`
- 系统生成一份 `赴约手册`
- 活动当天打开的是 `赴约提醒` 或 `赴约陪伴页`

这比“生成作战书”自然得多。

## 3. 按功能模块重新包装产品结构

下面这版是基于 `功能模块.docx` 做的重新命名和重包装，尽量保留你原本的功能结构，但把气质整体拉回“浪漫赴约”。

## 3.1 首页定位

原来不应该是：

- 创建作战书

建议改成：

- 开始这次赴约
- 为这场见面做个安排
- 把期待变成一份顺顺利利的赴约计划

首页场景仍保留：

- 演唱会 / 音乐节
- 球赛

但语气要像“进入一段旅程”，而不是“选择一个任务模式”。

## 3.2 演唱会 / 音乐节板块重命名

原功能模块中是：

- 衣
- 食
- 住
- 行
- 票
- 社交 / 物料

建议保留结构，但重命名为更有氛围感的表达：

### 衣 -> 穿搭与妆造

不是功能性“衣”，而是：

- 今天想穿什么
- 什么风格更适合这场
- 应援色 / 妆造 / 出片感

### 食 -> 补给与周边小店

比“食”更细腻，因为用户不是来点外卖，是想知道：

- 到场前吃什么方便
- 散场后去哪里续上情绪
- 附近有没有适合打卡的小店

### 住 -> 住得近一点 / 住得舒服一点

住宿模块不必太工具化，可以强调：

- 离场馆近
- 散场回去不折腾
- 和搭子住一起方便

### 行 -> 路线与出发节奏

不要只讲“交通”，要讲：

- 几点出发最从容
- 怎么去最顺
- 散场怎么回不狼狈

### 票 -> 门票与位置参考

“票务助手”太冷了，建议改成：

- 门票提醒
- 位置参考
- 开票前准备
- 补票/转票说明

### 社交 / 物料 -> 同场搭子与应援交换

不要写得像论坛模块，而要像：

- 找一起去的人
- 换物料
- 拼房拼车
- 认识同场的人

## 3.3 球赛板块重命名

原来功能模块里有：

- 行程规划
- 信息模块
- 现场观赛指南
- 社交与情绪宣泄
- 应援与物料

建议改成：

- 观赛安排
- 比赛信息
- 到场指南
- 同城球迷
- 应援与周边

其中“情绪宣泄”这个表达建议完全不要放在产品一级结构里，会显得粗糙，也容易把产品气质带偏。

## 3.4 通用能力重命名

### 赴约成功率 -> 这次安排得怎么样

不要做成冰冷评分系统，建议改成：

- 赴约安心度
- 行程顺滑度
- 这次安排得怎么样

我最推荐：

`赴约安心度`

因为它更符合用户心态。  
用户不是在打仗，而是在问：

- 我这样安排会不会太赶
- 到时候会不会慌
- 我现在放心了吗

### 防踩坑雷达 -> 提前提醒你

可替换为：

- 提前提醒你
- 这几件事先别忘
- 出发前记得看看

推荐：

`提前提醒你`

### 现场指挥中心 -> 今天这样去最顺

可替换为：

- 今天这样去最顺
- 今天的赴约提醒
- 当天陪伴页

推荐：

`今天这样去最顺`

这句话更像朋友在陪你，而不是系统在指挥你。

## 4. 重新定义产品主价值

原来的主价值偏向：

- 防踩坑
- 提高成功率
- 提前预判风险

这些没错，但太“功能正确”，不够“产品动人”。

新的主价值应该改成三层：

### 第一层：让赴约更安心

用户不用再到处搜：

- 能带什么
- 几点去
- 怎么回
- 住哪里

### 第二层：让赴约更有仪式感

用户不只是“去看一场活动”，而是在认真准备一场期待已久的见面。

### 第三层：让赴约更有陪伴感

无论是一个人去、和朋友去、还是找搭子去，产品都像在陪用户把这次安排好。

## 5. 新的产品一句话

我给你几版可选。

### 版本 1

把每一次期待已久的见面，安排成一场顺顺利利的赴约。

### 版本 2

从想去，到见面，这次赴约都替你提前想好了。

### 版本 3

帮你把演出、球赛和每一次期待，认真安排成一份赴约计划。

### 版本 4

不是冷冰冰的攻略，而是一份陪你安心赴约的贴心安排。

我最推荐版本 2 或版本 4。

## 6. 页面文案整体替换建议

### 首页

- 原：创建赴约作战书
- 改：创建赴约计划

### 结果页标题

- 原：AI 赴约作战书
- 改：你的赴约手册

### 评分模块

- 原：赴约成功率
- 改：赴约安心度

### 风险模块

- 原：防踩坑雷达
- 改：提前提醒你

### 时间线模块

- 原：跟着这份节奏走
- 改：这一天可以这样安排

### 清单模块

- 原：入场清单
- 改：出发前收一收

### 建议模块

- 原：临场建议
- 改：给你的贴心提醒

### 当天页

- 原：现场指挥中心
- 改：今天这样去最顺

## 7. 我建议你现在就定下来的方向

如果我们要把这个产品重新做对，我建议你直接采用下面这套统一命名：

- 产品定位：`一站式赴约小管家`
- 核心动作：`创建赴约计划`
- 核心产物：`赴约手册`
- 核心评分：`赴约安心度`
- 风险提醒：`提前提醒你`
- 当天页面：`今天这样去最顺`

这套语言比“作战书”更温柔，也更符合“赴约”本身的情绪价值。

## 8. 下一步怎么做

我建议下一步不要只改一句文案，而是直接整体改一轮：

1. 改产品命名体系
2. 改分享页和结果页文案
3. 改结果页模块标题与语气
4. 顺带把视觉继续往“浪漫赴约”收，而不是“炫彩后台”

这样页面、产品概念和功能模块才能统一。
