# 一站式赴约小管家 UI 设计文档 v2

## 1. 本轮新增结论

在 `ui_design_spec_v1.md` 的基础上，这一版重点新增 5 个方向：

1. 文字样式不能再“呆、硬、黑、粗暴”
2. 侧边栏要升级成真正的产品导航结构
3. 首页要把 `演唱会/音乐节赴约` 和 `球赛赴约` 做成明确的两大板块入口
4. 增加更有氛围感的动效与轮播内容
5. 不是只改首页，而是后续页面全部沿用这套系统继续统一

这意味着我们接下来不再只是“把页面变亮”，而是要正式建立一套：

- 适合赴约主题的排版语言
- 适合宽屏产品的导航体系
- 适合活动类产品的内容编排方式
- 适合浪漫、轻盈气质的动效系统

## 2. 文字风格问题

## 2.1 当前问题

你指出的非常准确：  
现在的标题虽然已经比之前亮了，但仍然有这些问题：

- 太硬
- 太黑
- 太“排版工具”
- 没有情绪弹性
- 不像高质量产品首页，更像放大后的默认粗体字

视觉上会给人一种：

- 信息很大
- 但不好看
- 缺乏节奏
- 没有呼吸感

## 2.2 新的文字方向

首页文字应从“硬核口号”改成“有呼吸感的品牌标题”。

### 标题风格建议

- 字重不一定非要最黑
- 允许更有节奏地分行
- 可以通过大小对比制造层次，而不是全靠超粗
- 保持大标题，但更讲究留白与节奏

### 标题气质关键词

- 轻盈
- 浪漫
- 呼吸感
- 有编辑感
- 像品牌海报，而不是系统标题

## 2.3 标题样式建议

### 推荐方案 A：强标题 + 柔副标题

主标题使用：

- 深紫灰色，不用纯黑
- 字重 `700 - 800`
- 行高更紧，但不要糊成一团
- 通过两行或三行切分形成节奏

副标题使用：

- 更浅的灰紫色
- 字号更大一点，不要太细碎
- 行距舒展

### 推荐方案 B：关键词高亮

比如：

- “浪漫一点”
- “安心一点”

这类词可用更柔和的粉紫 / 杏橙做高亮，而不是整段同色。

### 推荐方案 C：中英文层级分离

`YOUR DATE STARTS HERE` 可以保留，
但只作为氛围标签，不应抢主标题的重心。

建议：

- 更细
- 更浅
- 更有字距

## 2.4 文字颜色建议

避免：

- `#1A1A1A` 这种过硬纯深黑

建议改为：

- 主标题：`#2A2340`
- 次标题：`#6F6B7D`
- 英文眉标：`#9D8FA8`

## 3. 首页结构升级

## 3.1 当前首页结构的问题

虽然现在已经比第一版好，但仍有几个结构问题：

1. 侧边栏还只是“跳转目录”，不像真正产品导航
2. 首页还是偏“一张 landing page”
3. `演唱会/音乐节赴约` 和 `球赛赴约` 的区分不够强
4. 内容虽然多了，但没有形成真正的产品分区

## 3.2 首页应升级为“门户首页”

首页不应只是一个宣传首屏，而应成为整个产品的总入口。

建议采用：

`左侧产品导航 + 右侧主内容门户`

## 3.3 侧边栏改版建议

### 一级结构

侧边栏建议固定为两个主板块：

1. `演唱会 / 音乐节赴约`
2. `球赛赴约`

这是你这次提出的关键要求，我建议保留，而且这是对的。

原因：

- 用户进入后先选大场景
- 产品结构更清晰
- 比现在“模块堆在一起”更像真正平台

### 二级结构

在两个主板块下，再展开模块入口。

#### 演唱会 / 音乐节赴约

- 创建赴约计划
- 穿搭与妆造
- 补给与周边
- 住得近一点
- 路线与节奏
- 门票与位置
- 同场搭子

#### 球赛赴约

- 创建观赛安排
- 门票与看台
- 到场动线
- 散场返程
- 周边补给
- 同城球迷
- 应援与周边

### 侧边栏状态设计

状态建议：

- 默认：浅灰底
- Hover：淡粉紫或淡橙背景
- 当前选中：白底 + 彩色左侧高亮条

而不是：

- 纯文字列表
- 或全部做成同一批 chip

## 3.4 首页主内容区结构

首页主内容区建议分成 5 层：

### 模块 1：Hero 区

内容：

- 标题
- 副标题
- 快速开始面板

### 模块 2：现场轮播区

这里可以加入：

- 演唱会现场图
- 音乐节氛围图
- 球赛看台图

作用：

- 拉高情绪
- 让产品更像“赴约入口”
- 减少纯文字页面的呆板感

### 模块 3：快速问题 / 快捷入口

类似：

- 第一次一个人去演唱会怎么办
- 散场赶高铁怎么安排
- 想找住得近一点的地方

### 模块 4：按场景板块进入

这里不再只是 3 张普通卡，
而应该更像两个“产品专区”：

#### 演唱会 / 音乐节专区

- 更浪漫
- 更粉紫
- 更有灯海感

#### 球赛专区

- 更活力
- 更橙黄
- 更有动势

### 模块 5：最近安排 / 历史手册

保持产品连续性。

## 4. 轮播图设计建议

## 4.1 是否要加轮播图

建议加，但不能像老式 banner 轮播广告。

正确方式是：

`情绪型场景轮播`

也就是：

- 演唱会灯海
- 音乐节彩带
- 球赛现场看台

用作气氛烘托，不作为促销图。

## 4.2 轮播图区域适合放哪里

推荐两种：

### 方案 A：Hero 右下方作为大卡片轮播

优点：

- 更像现代平台首页
- 不抢主标题

### 方案 B：Hero 下方单独一行宽图轮播

优点：

- 更适合宽屏
- 情绪表达更强

我更推荐：

`方案 B`

因为你现在希望首页横向更满、内容更多，独立轮播带更合适。

## 4.3 轮播图视觉原则

- 大圆角
- 明亮
- 可轻微自动切换
- 不要带强烈广告横幅感

建议叠加信息：

- 场景名
- 一句简短提示

例如：

- 演唱会赴约：把灯海前后的节奏安排好
- 音乐节赴约：舞台切换和体力准备更轻松
- 球赛赴约：看台、动线、返程都更顺

## 5. 动效系统建议

## 5.1 总原则

动效要：

- 轻
- 柔
- 有质感

不能：

- 太炫
- 太跳
- 太游戏化

## 5.2 首页动效建议

### Hero

- 标题淡入上浮
- 副标题延迟出现
- 标签轻微错位入场

### 规划面板

- 初始淡入
- hover 时卡片轻微抬起

### 轮播图

- 平滑切换
- 不用夸张滑动

### 侧边栏

- 当前模块高亮条滑动
- hover 背景轻柔过渡

## 5.3 内页动效建议

### 创建页

- 表单分区顺序出现
- 聚焦输入框时边框染色

### 赴约手册页

- 模块依次出现
- 时间线节点渐入
- 卡片 hover 微微漂浮

## 6. 全站页面统一规划

你这次提得很对：  
不是只把首页改了就行，后续页面也要统一。

## 6.1 首页

关键词：

- 门户
- 引导
- 平台感
- 情绪感

## 6.2 创建页

关键词：

- 轻表单
- 像旅行规划工具
- 清爽
- 分区明确

### 创建页建议新增左侧边栏

例如：

- 基本信息
- 出发方式
- 在意的细节
- 特别备注

让用户有“正在一步步完成计划”的感觉。

## 6.3 赴约手册页

关键词：

- 文档感
- 卡片感
- 分享感
- 收藏感

### 建议增加右侧辅助栏

可包含：

- 快速跳转目录
- 一键分享
- 保存
- 返回创建页

## 6.4 我的安排页

关键词：

- 明亮
- 像作品集 / 行程库
- 卡片墙

### 设计方式

- 白底
- 大卡片
- 按场景筛选
- 最近创建 / 已收藏 / 已分享标签

## 7. 两大场景专区的视觉区分

## 7.1 演唱会 / 音乐节专区

视觉关键词：

- 灯海
- 粉紫
- 柔焦
- 浪漫
- 闪光点

推荐配色：

- 底：`#FFF7FB`
- 高光：`#FFB7C5`
- 辅色：`#C3A0FF`

## 7.2 球赛专区

视觉关键词：

- 热烈
- 阳光
- 活力
- 节奏感
- 场馆氛围

推荐配色：

- 底：`#FFF9F0`
- 高光：`#FF9F4A`
- 辅色：`#FFD966`

## 7.3 通用层

共用底层颜色：

- 白卡片
- 深灰紫文字
- 淡阴影

这样可以做到：

- 专区有区分
- 但全站仍然统一

## 8. 字体与排版 v2 建议

## 8.1 字体建议

首页标题不要再用“纯粗默认无衬线放大版”。

建议方向：

- 更圆润的无衬线
- 字重有层次
- 中文标题更有留白

即使不换字体文件，也要通过：

- 字重
- 字间距
- 行高
- 排列方式

来修正气质。

## 8.2 排版节奏

首页大标题建议：

- 2-3 行
- 留白明显
- 每行长度更均匀

不要：

- 一坨超重字堆在左边

## 8.3 可尝试的标题结构

例如：

`把这场见面`
`安排得更浪漫`
`也更安心`

这种结构会比现在更像设计，而不是系统排版。

## 9. 本轮明确不建议做的事情

1. 再回到深色主界面
2. 再用整屏暗色背景图压页面
3. 只换背景不改组件
4. 只做表面炫彩，不做结构升级
5. 所有模块都做成同样的卡片
6. 首页不做板块区分

## 10. 下一步执行建议

下一步不建议全站同时大改。  
建议分 3 步：

### Step 1：首页 v3

- 改标题排版
- 侧边栏升级为双板块导航
- 加轮播区
- 横向布局彻底铺开

### Step 2：创建页 v2

- 改为宽屏 + 左侧分区导航
- 改文字样式
- 改输入框与分组逻辑

### Step 3：赴约手册页 v3

- 改成更像“手册”的排版
- 加右侧目录
- 加更轻的动效

## 11. 本轮最终推荐

如果现在只定一句设计方向，我建议这样描述：

`像一个明亮高级的旅行规划平台，但服务的是演唱会、音乐节和球赛赴约。`

也就是说：

- 结构学旅游平台
- 情绪学活动产品
- 交互学 AI 规划工具
- 视觉学明亮生活方式产品
