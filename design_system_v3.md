# 赴约小管家 · 设计系统 V3

> 基于用户明确反馈，本版彻底放弃浅色卡片底色方案，全面转向成熟 OTA 风格。
> 白底高密度 + 蓝色主操作 + 六大模块彩色标识系统。

---

## 一、设计方向

用户明确选择的方向：

- **白底成熟 OTA 风格**（参考携程、Trip.com）
- **蓝色作为主操作色**
- **拒绝浅色幼稚感**
- **信息密度高、专业可信任**

## 三、六大模块 · 彩色标识系统

**核心原则**：模块卡片统一使用白底，通过左侧 4px 彩色竖条 + 彩色图标来区分，不再使用整卡浅色底色。

| 模块 | 标识色 | 色值 | 命名 | 图标风格 |
|------|--------|------|------|----------|
| 衣 | 珊瑚橙 | `#FF6B4A` | Coral | 衣架/服饰图标 |
| 食 | 琥珀金 | `#FF9500` | Amber | 餐饮/餐具图标 |
| 住 | 薄荷绿 | `#00C853` | Mint | 酒店/房屋图标 |
| 行 | 主蓝色 | `#2878FF` | Blue | 交通/路线图标 |
| 票 | 优雅紫 | `#7B61FF` | Purple | 票据/座位图标 |
| 社交 | 玫瑰粉 | `#FF4081` | Rose | 人物/对话图标 |

### 3.1 模块卡片结构

```
┌─────────────────────────────────────┐
│▌  图标  模块标题                        │  ← 左侧 4px 色条 + 图标 + 标题
├─────────────────────────────────────┤
│                                     │
│  内容区域（白底）                      │  ← 内容按模块特性用不同布局
│                                     │
└─────────────────────────────────────┘
```

### 3.2 模块布局节奏（差异化）

| 模块 | 布局方式 | 原因 |
|------|----------|------|
| **衣** | 左图右文 / 搭配卡片流 | 穿搭需要视觉展示 |
| **食** | 时间线三段式 | 入场前 → 在场 → 散场后 |
| **住** | 2×2 网格对比卡片 | 价格/位置/评价对比 |
| **行** | 纵向时间轴 | 出发 → 到场 → 散场 → 返程 |
| **票** | 信息摘要表 | 票档/入口/注意事项结构化 |
| **社交** | 对话气泡 + 人物卡片 | 搭子/会合的人际感 |

## 四、组件规范

### 4.1 卡片

- 背景：`#FFFFFF`
- 边框：`1px solid #E8E8E8`
- 圆角：`12px`
- 阴影：无（OTA 风格不用阴影，用边框分层）
- 内边距：`16px - 20px`

### 4.2 按钮

**主按钮**
- 背景：`#2878FF`
- 文字：`#FFFFFF`
- 圆角：`8px`
- 高度：`44px`
- Hover：`#1A66E6`

**次按钮**
- 背景：`#FFFFFF`
- 边框：`1px solid #E8E8E8`
- 文字：`#333333`
- Hover：背景 `#F5F7FA`

**文字按钮**
- 文字：`#2878FF`
- 无背景无边框

### 4.3 标签/Chip

- 背景：`#F5F7FA`
- 边框：`1px solid #E8E8E8`
- 圆角：`16px`（胶囊型）
- 文字：`#666666`
- 选中态：背景 `#2878FF`，文字 `#FFFFFF`

### 4.4 输入框

- 背景：`#FFFFFF`
- 边框：`1px solid #E8E8E8`
- 圆角：`8px`
- Focus：边框 `#2878FF`
- 高度：`44px`

## 六、与首页的关系

首页保持现有的粉紫暖光浪漫感（这是品牌入口）。

**内页从首页跳进来时**：
- 导航保持风格一致（胶囊型、轻量）
- 但内页整体转向更成熟、信息密度更高的风格
- 这种"从浪漫入口进入专业工具"的反差是可接受的，甚至是有益的

---

## 七、不再使用的设计

以下设计元素本版本明确废弃：

- ❌ 整卡浅色底色（如 `#FAECE7` 等）
- ❌ 大面积粉紫渐变背景
- ❌ 玻璃拟态/毛玻璃效果
- ❌ 圆角过大（超过 16px）
- ❌ 阴影浮层卡片
- ❌ 情绪化的大字标题（内页需要信息效率）

---

## 八、执行顺序

1. **先改 CSS 变量和基础色板**（全局替换）
